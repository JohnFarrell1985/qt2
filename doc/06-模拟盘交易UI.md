# 模拟盘交易 UI (src/webui/)

本地运行的自包含**模拟盘 (paper trading)** 交易终端, 电脑端与手机端浏览器均可访问。
界面参考同花顺远航版**交易终端**(交易页, 非行情页), 遵循 A 股习惯 **红涨绿跌**。

不依赖 QMT 终端 —— 纯本地模拟撮合, 用于练习/验证选股与交易节奏。

## 启动

```bash
python -m src.webui                 # 默认 0.0.0.0:8001
python -m src.webui --port 8001     # 指定端口
python -m src.webui --reload        # 开发热重载
```

- 电脑访问: `http://127.0.0.1:8001`
- 手机访问: 与电脑同一局域网, 浏览器打开 `http://<电脑IP>:8001`(启动时会打印局域网地址)

## 模块结构

| 文件 | 职责 |
|------|------|
| `server.py` | FastAPI 服务 + REST 接口 + 鉴权 + 多用户账户缓存 + 托管前端 |
| `paper_engine.py` | 模拟盘账户与撮合引擎 (按用户名隔离, 线程安全) |
| `store.py` | 持久化存储 (`paper_users`/`paper_accounts`/`paper_trades` 表; DB 与内存两种实现) |
| `auth.py` | 用户鉴权 (PBKDF2 口令哈希 + 内存会话令牌) |
| `quotes.py` | 行情报价 (QuoteProvider) + 日线/交易日历 (DayBarProvider) |
| `selection_service.py` | 每日选股/选基服务 (按 strategy_id 调度, 与 QMT 无关) |
| `src/selection/strategy.py` | 选股策略抽象基类 + 注册表 |
| `src/selection/strategies/` | 各策略具体实现 (当前: `bull_launch`) |
| `__main__.py` | uvicorn 启动入口 |
| `static/` | 响应式单页前端 (index.html / style.css / app.js) |

选股复用各策略模块内的实现; 启动突破使用 `ma_screener` (股票) 与 `etf_screener` (ETF)。

## 多用户 (v6.0)

- **注册**: 仅需 **用户名 + 口令** (其余无需)。默认用户 **`root` / `1234`** (首启自动创建)。
- **登录**: 校验口令后签发内存会话令牌 (进程级, 服务重启需重新登录)。前端令牌存 `localStorage`,
  经请求头 `X-Auth-Token` 携带; 账户类接口未登录返回 401。
- **数据隔离**: 每个用户的账户状态与成交流水按 `username` 隔离, 分别存于 `paper_accounts` /
  `paper_trades` 表。
- **调整模拟资金**: 用户可随时把可用现金设为任意 **非负** 数值 (存/取); 出入金同步平移
  `initial_capital` 与当日起始资产, 使盈亏口径不受出入金影响。
- **删除成交记录**: 用户可逐条删除 **自己的** 成交流水 (仅删记录, 不回滚已产生的资金/持仓变动)。

## 存储与数据安全

- 模拟盘全部数据仅落在 **三张全新表** (与现有业务表分离, 同一 DB 内):
  `paper_users` / `paper_accounts` (JSONB 引擎状态) / `paper_trades` (逐条成交)。
- 行情/日线读取一律经 **只读会话** (`get_session(readonly=True)`); 写操作只发生在 `paper_*` 表。
  **模拟盘永不修改行情等真实业务表** (如 `stock_daily`)。
- 无 DB 时自动回退内存存储, 服务仍可运行 (数据不持久化)。

## 交易规则

- **手续费 (华泰证券)**: 复用 `src/backtest/fees.py`
  - A 股: 佣金万1.15(最低5元) + 卖出印花税千0.5 + 沪市过户费万0.2
  - 港股通: 佣金万3(最低5港元) + 印花税千1 + 交易费/交易征费/会财局征费/股份交收费
- **T+1 / T+0 (以交易日为准)**: A 股当日买入次日可卖; **港股通 / 可转债 / 跨境 ETF** 当日可回转 (T+0)。
  分类依据 `src/common/asset_types.py` 的 `infer_asset_type`。
- **卖出校验**: 无持仓不可卖; 可用不足(T+1 未解冻)不可卖。
- **委托方式**: 限价单、市价单
- **申报单位**: 按板块自动规整 (科创板200股起、可转债10张等), 见 `src/trading/market_rules.py`

## 按交易日回放 (手动回测, v5.2)

模拟盘以 **当前交易日** (`state.trade_date`) 驱动, 可切换到库中任意有数据的历史交易日, 实现「手动导入历史 + 类回测」的下单练习:

- **交易日历**: 顶部「交易日」条提供上一/下一交易日按钮与日历弹层, 日历中仅本地库有数据的交易日可点选。
  新账户默认对齐到库中最新交易日。
- **当日价格区间**: 输入代码后, 从本地 PostgreSQL 日线 (`stock_daily`/`etf_daily`/`cb_daily`) 读取
  当日 **开/最高/最低/收**。买卖价格必须落在 **[最低, 最高]** 区间内, 面板提供「最低/开/收/最高」一键选价。
- **撮合规则 (日回放模式)**:
  - **跟盘 (模拟日 = 库内最新交易日)**: 收盘后选股/下单, 委托一律 **次日** 生效; 同步日 K 后
    按「最低价 < 挂单价」(买) 撮合; 生效日无 K 线则继续挂单。
  - **历史回放 (模拟日 < 最新日)**: 限价单可在当日线价区间 [最低, 最高] 内即时成交 (补录)。
  - 市价单: 按当日 **收盘价** 成交。
- **换日解冻**: 切换交易日即触发 T+1 解冻 (隔日持仓变为可卖) 并重置「当日盈亏」基准。
- **估值**: 持仓按当前交易日 **收盘价** 估值。
- **降级**: 若某代码在当前交易日无日线数据, 退回「实时报价」撮合 (见下)。

## 每日选股 / 选基 (v6.1)

「选股」视图可按**已注册**策略对全市场做初筛, 勾选后批量挂**次日撮合**的买入单。

### 架构 (策略与选股解耦)

```
Web UI (selection_service)  ──strategy_id──▶  strategy.py 注册表
                                                    │
                                                    ▼
                                         strategies/bull_launch.py  (SelectionStrategy 子类)
                                                    │
                                    ┌───────────────┴───────────────┐
                                    ▼                               ▼
                            ma_screener (股票)              etf_screener (ETF)
```

- **选股服务层** (`selection_service.py`): 只按用户选择的 ``strategy_id`` 调度, **不**直接 import 筛选逻辑; 与 QMT 无关。
- **策略基类** (`src/selection/strategy.py`): ``SelectionStrategy`` 定义 ``screen()`` / ``catalog_entry()``; 注册表 ``@register_strategy``。
- **策略实现** (`src/selection/strategies/`): 每个策略一个模块, 继承基类并实现具体逻辑。当前仅 **启动突破 (`bull_launch`)** 已注册。
- **参数预设** (`config/strategies/*.json`): 可配置默认值; 仅有 JSON、尚未实现 Python 类的策略 (如 `bear_rebound`) **不会出现在 UI**。
- **UI 参数**: 浏览器 ``localStorage`` 覆盖 JSON 默认, 不修改全局 ``settings``。

### 功能要点

- **选股 / 选基**: 「每日选股」跑股票, 「每日选基」跑 ETF; 后台线程 + 状态轮询。
- **挂单撮合**: 以账户当前交易日 ``D`` 为基准, 批量挂单于 ``D+1`` 生效; 当日最低价 < 挂单价则成交, 成交价 = ``min(挂单价, 当日最高)``。
- **新增策略**: 在 ``strategies/`` 添加模块并 ``@register_strategy``, 可选添加 ``config/strategies/{id}.json`` —— **无需改** ``selection_service.py``。

## 行情来源 (无日线时的降级)

`quotes.py` 依次尝试: `stock_realtime` 快照 → `stock_daily`/`etf_daily`/`cb_daily` 最新收盘 →
(可选) akshare 实时 → 上次已知价。无任何来源时可用 `POST /api/quote/manual` 手动置价。

## 模块结构 (行情/日线)

| 文件 | 职责 |
|------|------|
| `quotes.py::QuoteProvider` | 实时/最新价 (多级降级) |
| `quotes.py::DayBarProvider` | 按 (代码, 交易日) 读取日线 OHLC + 交易日历/相邻交易日 (ETF/可转债按带后缀代码匹配 `etf_daily`/`cb_daily`, ETF 名称取自 `etf_info`) |

## REST 接口

除鉴权与公共接口外, 账户/交易/交易日接口均需在请求头携带 `X-Auth-Token`。

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/register` | 注册 `{username, password}` → `{token, username}` |
| POST | `/api/login` | 登录 `{username, password}` → `{token, username}` |
| POST | `/api/logout` | 注销当前会话 |
| GET | `/api/me` | 当前登录用户 |
| GET | `/api/state` | 账户全景 (资产/持仓/委托/成交, 含 `trade_date`、实时估值与挂单撮合) |
| GET | `/api/quote?code=` | 单只标的最新价 |
| GET | `/api/fee_preview?code=&price=&quantity=&direction=` | 预估手续费 |
| GET | `/api/search?kw=` | 代码/名称联想 (依赖 `stocks` 表, 只读) |
| GET | `/api/trade_date` | 当前交易日 + 库中最新交易日 |
| POST | `/api/trade_date` | 切换交易日 `{date}` (返回账户快照) |
| GET | `/api/trade_date/step?direction=prev|next` | 切换到相邻交易日 |
| GET | `/api/calendar?start=&end=` | 区间内有数据的交易日列表 (供日历渲染) |
| GET | `/api/day_range?code=&date=` | 某交易日日线 (开/高/低/收/昨收/涨跌幅) |
| POST | `/api/sync/kline` | 启动 QMT 日K 同步 `{days_back?, concurrency?, source?}` (默认 15 日 / qmt / 4 并发) |
| GET | `/api/sync/status` | 同步任务状态 (etf/stock 阶段、行数、结算结果) |
| POST | `/api/order` | 下单 `{code, direction, quantity, price, price_type}` |
| POST | `/api/cancel` | 撤单 `{order_id}` |
| POST | `/api/capital` | 调整模拟资金 `{cash}` (非负, 返回账户快照) |
| POST | `/api/trade/delete` | 删除本用户成交记录 `{trade_id}` |
| POST | `/api/quote/manual` | 手动置价 `{code, price}` |
| POST | `/api/reset` | 重置账户 `{initial_capital}` |
| GET | `/api/strategies` | 策略列表 + 每策略可编辑参数(schema+默认值) |
| POST | `/api/select` | 启动后台选股/选基 `{kind: stock\|etf, strategy_id, date?, params}` |
| GET | `/api/select/status?kind=` | 任务状态 (running/elapsed/count/error) |
| GET | `/api/select/result?kind=` | 当前选股记录 (策略/参数/结果/run_id) |
| GET | `/api/select/history?kind=` | 历史记录列表 (最近 40 条) |
| GET | `/api/select/history/{run_id}` | 单条历史详情 (含参数与完整列表) |
| GET | `/api/select/inst-holders/status?kind=` | 机构家数异步抓取状态 |
| GET | `/api/select/inst-holders?kind=` | 机构家数结果 `{items: {code: {...}}}` |
| POST | `/api/picks/order` | 批量选股挂单 `{kind, items:[{code, price, quantity}]}` (次日生效) |

## 界面

- **登录/注册**: 首屏遮罩, 仅用户名 + 口令; 记住令牌, 顶部「退出」注销
- **顶部资产条**: 用户名 / 总资产 / 当日盈亏 / 总盈亏 / 可用资金 / 持仓市值 / 改资金 / 重置 / 退出
- **交易日条**: 上一/下一交易日、当前交易日、日历弹层、**「同步数据」**(QMT 增量日K + 挂单结算)、库中最新交易日提示
- **交易面板**: 买入/卖出切换、代码联想、当日价格区间与一键选价、价格±步进(按板块 tick)、数量±步进与快捷仓位(全仓/½/⅓/¼)、市价(按收盘)开关、金额与费用预估
- **持仓 / 委托 / 成交** 三列表: 持仓行可一键转卖出、挂单可撤单、成交记录可逐条删除
- **选股视图**: 策略下拉 + **全量参数表** (始终展示) + 每日选股/每日选基按钮 + 进度提示;
  **股票/ETF 结果分 tab**, 各自保留最近一次结果 (PostgreSQL `paper_selection_runs`), 再次进入无需重跑;
  左侧**历史记录**可回溯任意一次 (点击恢复该次参数与列表); 「当前」回到最新一次。
  结果表先返回列表 (股票机构家数显示加载中转圈, 后台东财/新浪多源异步抓取后自动刷新并回写 DB); ETF 不显示机构家数。含勾选、评分/分层、收盘、挂单价、数量与理由, 底部「批量挂单」。交易日条右侧「每日选股 ▸」进入。
- **手机端**: 底部导航(交易/选股/持仓/委托/成交), 单屏切换; 桌面端左右分栏, 选股视图铺满
