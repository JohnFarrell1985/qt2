# 数据模块 (src/data/)

## 概述

负责与迅投 QMT 对接，下载并同步行情、财务、因子数据到 PostgreSQL。  
核心设计: **分批下载 + 异步等待 + 增量续传 + 限流保护**，确保 5000+ 只 A 股数据稳定采集。

## 模块结构

| 文件 | 职责 |
|------|------|
| `qmt_client.py` | 封装 xtquant SDK 全部免费版接口 (30+ API) |
| `download_engine.py` | **下载引擎**: 分批、限流、重试、断点续传、异步等待 |
| `market_data.py` | 日线/分钟线/指数数据同步 |
| `factor_data.py` | 因子元信息管理 + 因子数据同步 |
| `financial_data.py` | 财务报表/指标同步 |
| `sync.py` | 统一同步调度 (全量/增量/按需) |
| `models.py` | SQLAlchemy ORM 模型定义 |
| `universe_provider.py` | 统一标的池提供者 (A股/ETF/可转债/港股通) |
| `schemas.py` | Pandera 数据质量 Schema (stock_daily / etf_daily / cb_data) |
| `quality.py` | 数据质量检查器 (Schema 校验 + 连续性 + Z-score 异常检测) |

## QMT API 接口一览

所有接口均参照 [xtquant 官方文档](http://dict.thinktrader.net/nativeApi/xtdata.html)。

### 行情数据

```python
# 板块列表
client.get_sector_list() -> List[str]

# 板块成分股 (常用: '沪深A股', '沪深300', '中证500')
client.get_stock_list_in_sector("沪深A股") -> List[str]

# 批量下载历史K线到本地 (异步, 带进度回调)
client.download_history_data2(stock_list, period="1d", start_time="20160101", end_time="20260402", callback=fn)

# 单股下载 (同步, 用于多进程场景)
client.download_history_data("000001.SZ", period="1d", start_time="20160101")

# 读取已下载的本地数据 (速度快, 不走网络)
client.get_local_data(stock_list, period="1d") -> {code: DataFrame}

# 新版行情接口 (推荐)
client.get_market_data_ex(stock_list, period="1d") -> {code: DataFrame}

# 全推行情 (高效批量订阅, 替代单股订阅)
client.subscribe_whole_quote(["SH", "SZ"], callback=fn) -> seq

# 最新交易日全推K线
client.get_full_kline(stock_list, period="1m", count=1)
```

## 下载引擎 (DownloadEngine)

### 为什么需要下载引擎?

直接调用 `download_history_data2` 有以下坑 (来自社区实测):

| 问题 | 说明 |
|------|------|
| **实际异步** | 官方文档说同步, 但实际异步执行, 下载后立即 get 可能拿不到数据 |
| **批量过载** | 5000只一次性下载会导致 QMT 数据服务卡死或超时 |
| **无限流** | 连续高频请求可能被服务端限速 |
| **无续传** | 如果中途断开, 需要全部重来 |
| **内存爆炸** | 一次性读取 5000 只股票的分钟线数据到内存 |

### 引擎架构

```
                     DownloadEngine
                          │
            ┌─────────────┼─────────────┐
            ▼             ▼             ▼
      download_kline  download_financial  get_local_kline_batched
            │             │                    │
     ┌──────┴──────┐     ...              (生成器, 分批读取)
     │  split 500  │
     │  per batch  │
     ├─────────────┤
     │ Batch 1     │──► download_history_data2 + Event.wait()
     │   ↓ pause   │                               │
     │ Batch 2     │──► download_history_data2 + Event.wait()
     │   ↓ pause   │                  ▲            │
     │ Batch N     │    retry on fail ─┘    callback: finished==total → set Event
     └─────────────┘
```

### 关键配置 (.env)

```bash
DL_BATCH_SIZE=500      # 每批股票数 (社区实测最稳)
DL_BATCH_PAUSE=2.0     # 批间暂停秒数 (防限流)
DL_RETRY_COUNT=3       # 失败重试次数
DL_RETRY_DELAY=5.0     # 重试间隔基数 (指数退避)
DL_TIMEOUT=600         # 单批超时秒数
```

### 各周期默认数据范围

| 周期 | 环境变量 | 默认起始 | 约数据量 (5000只) | 说明 |
|------|----------|----------|-------------------|------|
| 日线 `1d` | `DL_START_1D` | 20160101 | ~1250 万条 | 10年, 因子建模主力数据 |
| 周线 `1w` | `DL_START_1W` | 20160101 | ~250 万条 | 量很小 |
| 5分钟 `5m` | `DL_START_5M` | 20230101 | ~1.8 亿条 | 3年, 分钟级回测 |
| 15分钟 `15m` | `DL_START_15M` | 20230101 | ~6000 万条 | 3年 |
| 30分钟 `30m` | `DL_START_30M` | 20230101 | ~3000 万条 | 3年 |
| 1小时 `1h` | `DL_START_1H` | 20230101 | ~1500 万条 | 3年 |
| 1分钟 `1m` | `DL_START_1M` | 20250101 | ~3 亿条 | **仅1年! 数据量巨大** |
| tick | `DL_START_TICK` | 20260301 | 极大 | 仅最近1个月 |

> **提示**: 1 分钟线 5 年约 15 亿条, 需 TB 级存储, 请根据实际需求调整。

### 断点续传原理

```python
# incrementally=None 时, xtdata 自动判断:
# - 本地已有数据 → 从最后一条之后开始下载 (增量)
# - 本地无数据   → 从 start_time 开始下载 (全量)
# 相当于原生的断点续传能力

engine.download_kline(stock_list, "1d", incremental=True)   # 推荐: 自动续传
engine.download_kline(stock_list, "1d", incremental=False)  # 强制全量重下
```

## 数据同步策略

### 全量同步

```python
DataSyncManager().full_sync(
    start_date="",          # 空 = 各周期使用默认值
    sync_minute=True,
    minute_periods=["5m", "1m"],
    incremental=True,       # 增量续传
)
```

流程:
1. `download_sector_data` → 板块分类
2. `download_holiday_data` → 节假日
3. `download_index_weight` → 指数权重
4. `sync_stock_list` → A股列表 + 合约详情
5. 分批 download + sync 日线 (10批 × 500只)
6. 分批 download + sync 分钟线 (按周期)
7. download + sync 指数日线 (8支指数)
8. 分批 download + sync 财务报表
9. 分批 download + sync 财务指标
10. 分批 download + sync 因子数据

### 增量同步

```python
DataSyncManager().incremental_sync(days_back=5)
```

每日 17:00 定时执行, 仅同步最近 5 天数据。

### 按需同步

```python
DataSyncManager().sync_specific(
    stock_list=["000001.SZ", "600519.SH"],
    data_types=["daily", "minute_5m", "factors"],
    start_date="20250101",
)
```

### API 触发

```bash
# 全量同步 (后台执行)
POST /api/data/sync/full?sync_minute=true&minute_periods=5m,1m&incremental=true

# 增量同步
POST /api/data/sync/incremental?days_back=5
```

## UniverseProvider 统一标的池接口

将 `UniverseManager` (PIT 回测侧) 和 `InstrumentPoolManager` (策略侧) 统一为抽象接口，每种资产类型都有对应的 PIT 提供者，与 `TradingRule` 联动。

| Provider | 资产类型 | 数据来源 |
|----------|---------|---------|
| `AStockUniverseProvider` | A 股主板/科创/创业/北交 | `UniverseManager` PIT 查询 |
| `ETFUniverseProvider` | 境内 + 跨境 ETF | `InstrumentPoolManager` |
| `CBUniverseProvider` | 可转债 | `InstrumentPoolManager` |
| `HKConnectUniverseProvider` | 港股通 | `InstrumentPoolManager` |
| `CompositeUniverseProvider` | 组合式委托 | 注册多个子 Provider |

```python
from src.data.universe_provider import CompositeUniverseProvider

provider = CompositeUniverseProvider.create_default()
codes = provider.get_codes(trade_date=date(2025, 6, 1))
asset_type = provider.get_asset_type("688001")  # → AssetType.A_STOCK_STAR
```

## 数据质量监控 (Pandera Schema + DataQualityChecker)

入库前/回测前的三层校验，防止脏数据污染因子和模型。

### Schema 校验 (`src/data/schemas.py`)

基于 Pandera 定义，覆盖 stock_daily / etf_daily / cb_data 三种数据:

| 校验层 | 规则 | 示例 |
|--------|------|------|
| **字段类型** | code: str, trade_date: datetime, OHLCV: float | `Column(float, Check.gt(0))` |
| **业务规则** | high >= low, high >= open, high >= close | `Check(lambda df: (df["high"] >= df["low"]).all())` |
| **非空约束** | trade_date 不可为 null | `nullable=False` |

### DataQualityChecker (`src/data/quality.py`)

| 方法 | 功能 | 输出 |
|------|------|------|
| `validate_schema(df, "stock_daily")` | Pandera schema 校验 | `{"valid": bool, "errors": [...]}` |
| `check_continuity(df)` | 交易日连续性检查 (gap > 5 天标记) | `[{"gap_start", "gap_end", "gap_days"}]` |
| `detect_anomalies(df, "close", z_threshold=4)` | Z-score 异常值检测 | 异常行索引 |
| `full_check(df)` | 以上三项一键执行 | `{"schema": {}, "gaps": [], "anomalies": {}}` |

配置参数 (`.env`): `DQ_Z_THRESHOLD=4.0`, `DQ_MAX_PCT_CHANGE=22.0`
