# 运维部署手册

## 1. Docker 部署

### 1.1 前置条件
- Docker 20.10+
- Docker Compose v2
- PostgreSQL 已部署 (或使用外部实例)

### 1.2 构建与启动

```bash
# 构建镜像
docker compose build

# 启动服务
docker compose up -d

# 查看日志
docker compose logs -f qt-api
```

### 1.3 docker-compose.yml

```yaml
services:
  qt-api:
    build: .
    ports:
      - "8012:8012"
    env_file:
      - .env
    volumes:
      - ./models:/app/models
      - ./macro_env.json:/app/macro_env.json
    command: uvicorn src.api.main:app --host 0.0.0.0 --port 8012

  qt-scheduler:
    build: .
    env_file:
      - .env
    volumes:
      - ./models:/app/models
      - ./macro_env.json:/app/macro_env.json
    command: python -c "from src.api.scheduler import start_scheduler; start_scheduler(); import time; time.sleep(999999)"
```

### 1.4 Dockerfile

```dockerfile
FROM python:3.10-slim
WORKDIR /app
COPY pyproject.toml .
RUN pip install uv && uv sync --no-dev
COPY . .
EXPOSE 8012
CMD ["uvicorn", "src.api.main:app", "--host", "0.0.0.0", "--port", "8012"]
```

## 2. 环境变量

完整参数及详细注释见项目根目录 `.env` 文件 (新部署请先复制 `.env.example` 模板)。下表列出核心项:

| 变量 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `DATABASE_URL` | 否 | `postgresql://localhost:5432/qt_quant` | PostgreSQL 连接串 |
| `QMT_MINI_PATH` | 是 | — | MiniQMT userdata_mini 路径 |
| `QMT_ACCOUNT_ID` | 交易时 | — | QMT 交易账户 |
| `API_HOST` | 否 | 0.0.0.0 | API 监听地址 |
| `API_PORT` | 否 | 8012 | API 端口 |
| `LOG_LEVEL` | 否 | INFO | 日志级别 |
| `DL_BATCH_SIZE` | 否 | 500 | 下载引擎每批股票数 |
| `DL_BATCH_PAUSE` | 否 | 2.0 | 下载批间暂停秒数 |
| `DL_TIMEOUT` | 否 | 600 | 单批下载超时秒数 |
| `TRADING_PAPER_MODE` | 否 | true | 模拟盘模式 (false=实盘) |

## 4. 数据下载运维

### 4.1 首次部署数据初始化

首次部署需要执行全量数据同步, 详细操作步骤见 [用户手册 — 数据下载攻略](08-用户手册.md#6-数据下载攻略)。

```bash
# 通过 API 触发 (后台执行)
curl -X POST "http://localhost:8012/api/data/sync/full?incremental=true&sync_minute=true&minute_periods=5m"
```

**耗时估算**:
- 仅日线 + 财务: 30~60 分钟
- 含 5分钟线: 2~4 小时
- 含 1分钟线: 4~8 小时

### 4.2 磁盘监控

MiniQMT 数据缓存目录 (`QMT_MINI_PATH`) 占用空间会持续增长:

| 数据类型 | 预估空间 |
|----------|----------|
| 全A日线 10年 | 5~10 GB |
| 全A 5分钟线 3年 | 50~100 GB |
| 全A 1分钟线 1年 | 100~200 GB |
| 财务 + 基础数据 | 2~5 GB |

建议监控磁盘使用率, 超过 80% 时告警。

### 4.3 下载失败自动恢复

下载引擎内置断点续传, 如因网络中断或 MiniQMT 重启导致失败:
1. 确认 MiniQMT 客户端已重新启动
2. 重新触发相同的同步命令 (自动从断点续传)
3. 检查日志确认恢复正常

## 5. 定时任务

| 任务 | 时间 | 说明 |
|------|------|------|
| 增量数据同步 | 每天 17:00 | 同步最近5天数据 (含断点续传) |
| 持仓快照 | 每天 15:30 | 记录收盘持仓 |
| 模型重训 | 每4周 | 重新训练 LGB 模型 |

## 7. 扩展部署

### MiniQMT 部署
MiniQMT 必须在 Windows 环境运行。建议:
- 在 Windows 服务器部署 MiniQMT 客户端
- API 服务可部署在 Linux (通过网络访问 QMT)
- 或全部在 Windows 服务器上用 Docker Desktop 运行

### 多实例
- API 服务可水平扩展 (无状态)
- Scheduler 只能单实例 (避免重复定时任务)
