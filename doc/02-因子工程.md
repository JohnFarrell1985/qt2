# 因子工程模块 (src/factor/)

## 概述

因子工程处理链: 原始数据 → 因子计算 → 预处理 → 单因子检验 → 因子池。

因子是量化投资的"特征"——描述股票某个维度的数值指标。好的因子能预测未来收益率。

## 因子预处理流水线

原始因子值不能直接使用——不同因子量级差异大(如 ROE=15% vs 总资产=100亿)，且包含异常值。

```
原始因子值
    │
    ▼ Step 1: winsorize (MAD 去极值, ±3σ)
    │   消除异常值对模型的干扰
    │
    ▼ Step 2: standardize (Z-score 标准化)
    │   统一量级: 均值=0, 标准差=1
    │
    ▼ Step 3: neutralize (行业+市值中性化, OLS 回归残差)
    │   剥离行业和市值效应, 只保留因子本身的选股能力
    │
    ▼ 标准化后的因子值 → 可用于打分和 ML 训练
```

### 去极值 (MAD)

使用中位数绝对偏差 (MAD) 替代标准差，对异常值更稳健：

```
MAD_e = 1.4826 × MAD
上界 = median + 3 × MAD_e
下界 = median - 3 × MAD_e
超过上下界的值 → 截断到边界
```

为什么用 MAD 而非标准差? 标准差本身受异常值影响——如果有一只股票 ROE=500%，标准差会被拉得很大，导致去极值失效。MAD 基于中位数，天然抗异常值。

### 标准化 (Z-Score)

```
z = (x - mean) / std
```

使每个因子在同一截面上均值为 0、标准差为 1。这样：
- 不同因子可以直接相加 (Tier 2 打分)
- ML 模型训练更稳定

### 中性化

通过 OLS 回归消除行业和市值效应：

```
factor = β₀ + β₁×industry_dummies + β₂×ln(market_cap) + ε
中性化因子 = ε (残差)
```

**什么时候需要中性化**:
- Tier 2 打分策略如果不中性化，可能选出来全是大市值银行股 (因为银行 ROE 高且市值大)
- ML 策略通常不需要中性化 (模型自己能学到市值和行业的影响)

`.env` 参数: `STRAT_SCORE_NEUTRALIZE=false` (默认关闭，按需开启)

## 单因子检验

在把因子放入策略之前，必须验证它对收益率有预测能力。

| 指标 | 阈值 | 计算方式 | 说明 |
|------|------|---------|------|
| IC 均值 | > 0.02 | Spearman Rank IC: corr(因子排名, 未来收益排名) | 最核心指标 |
| ICIR | > 0.3 | IC均值 / IC标准差 | 衡量 IC 的稳定性 |
| IC 正向率 | > 55% | IC > 0 的占比 | 因子不能时好时坏 |
| 分组单调性 | G1 < G5 | 按因子值分 5 组，看收益是否单调 | 直观验证 |

### IC 检验解读

```
IC = 0.05:  优秀因子 (A 股市场 IC 0.03~0.05 就很好了)
IC = 0.02:  及格因子 (可用但不强)
IC = 0.01:  噪音 (不建议使用)
IC = -0.03: 反向因子 (取负值后使用)
```

### 分组回测

将全市场按因子值分为 5 组 (G1=最低, G5=最高):

```
如果因子有效 (正向):
  G1    G2    G3    G4    G5
  -2%   0%   +1%   +3%   +5%    → 单调递增 ✓

如果因子无效:
  G1    G2    G3    G4    G5
  +1%   -1%   +2%   0%   +1%    → 杂乱无章 ✗

如果因子反向有效:
  G1    G2    G3    G4    G5
  +4%   +2%   0%   -1%   -3%    → 单调递减 → 取负值使用
```

## 因子与策略的对接

### Tier 1 规则策略的隐式因子

规则策略虽然不显式使用因子，但其逻辑等价于单因子选股：

| 策略 | 等价因子 | 说明 |
|------|---------|------|
| 动量 | mom_20 | lookback 收益率排序 |
| 反转 | -mom_10 | 负动量 (跌幅排序) |
| 低波红利 | -vol_60, div_yield | 负波动率 + 高股息 |
| 均线突破 | MA_cross | 金叉/死叉二值信号 |
| 行业轮动 | industry_mom | 行业平均动量 |

### Tier 2 多因子打分的因子选择

推荐因子组合 (按投资风格):

| 风格 | 推荐因子 | 说明 |
|------|---------|------|
| 价值型 | eps, bps, roe, net_margin | 低估值 + 高盈利 |
| 成长型 | revenue_growth_yoy, net_profit_growth_yoy, eps | 高增速 |
| 动量型 | mom_20, mom_60, volume_ratio, rsi_14 | 趋势追随 |
| 质量型 | roe, gross_margin, gear_ratio | 优质公司 |
| 均衡型 | roe, mom_20, vol_20, eps, turnover_avg_20 | 多维度平衡 |

**实操建议**:
1. 先对 10~20 个候选因子做 IC 检验
2. 剔除 IC < 0.02 的
3. 去除高相关因子 (|corr| > 0.7 保留 IC 高的)
4. 5~10 个因子组合做多因子打分
5. 或者交给 ML 自动迭代选择

### Tier 3 ML 策略的因子输入

ML 策略使用全部通过 IC 初筛的因子作为特征 (通常 20~50 个)。

```python
# AutoIterateEngine 会自动:
# 1. 从 400+ 因子中 IC 初筛 → 30 个
# 2. LightGBM 训练 → feature_importances_
# 3. 淘汰底部 20% → 引入新候选
# 4. 循环迭代至收敛
```

## 常见问题

### Q: 该用多少因子?
- Tier 2 打分: 5~10 个 (太多会互相抵消)
- Tier 3 ML: 20~50 个 (ML 能处理高维度)
- 关键不是数量而是质量——IC < 0.02 的因子加再多也没用

### Q: 因子失效了怎么办?
A 股因子的有效性会随市场风格漂移。建议:
1. 每月重新做 IC 检验
2. ML 模型每 4 周重训
3. 观察因子频率统计——如果某因子在迭代中频率下降，可能在失效

### Q: 如何添加自定义因子?
在 `factor_calc.py` 中添加计算函数，然后在 `FACTOR_CATALOG` 注册即可。计算函数接收日线 DataFrame，返回 Series。

## BaseFactor 抽象基类 + FactorRegistry

因子系统采用"因子一等公民"架构: 所有因子继承 `BaseFactor` ABC, 由 `FactorRegistry` 统一注册管理。

### BaseFactor

```python
from src.factor.base import BaseFactor

class MyFactor(BaseFactor):
    @property
    def name(self) -> str: return "my_custom_factor"

    @property
    def category(self) -> str: return "momentum"

    def compute(self, df: pd.DataFrame) -> pd.Series:
        return df["close"].pct_change(20)
```

每个因子携带统一元数据: `name`, `category`, `version`, `data_source`, `lookback_days`。

### FactorRegistry

```python
from src.factor.base import FactorRegistry

FactorRegistry.register(MyFactor())
all_factors = FactorRegistry.list_all()
momentum_factors = FactorRegistry.list_by_category("momentum")
factor = FactorRegistry.get("my_custom_factor")
```

## Alpha158 因子计算器

参考 Qlib Alpha158 handler 实现的 6 大类量价因子 (`src/factor/alpha158.py`):

| 类别 | 因子数 | 代表因子 | 说明 |
|------|--------|---------|------|
| KBAR (K线形态) | ~16 | `(close-open)/open`, `(high-low)/close` | 日内价格结构 |
| PRICE (价格趋势) | ~18 | `close/close_5d`, `close/close_20d` | 多周期价格动量 |
| VOLUME (成交量) | ~18 | `vol/vol_5d`, `vol_5d/vol_20d` | 量能变化 |
| STD (波动率) | ~6 | `std_5d`, `std_20d`, `std_60d` | 多周期波动 |
| RSRS (阻力支撑) | ~6 | `high_max_5d/close`, `low_min_5d/close` | 价格极值 |
| CORR/COV (相关性) | ~94 | `corr(close, vol, 5d)` | 量价相关性 |

## 自动因子筛选管线 (AutoFactorScreen)

`src/factor/auto_screen.py` 实现三步自动筛选:

1. **IC/ICIR 阈值筛选**: IC > 0.03 且 ICIR > 0.3 且正比例 > 55%
2. **因子衰减检测**: 滚动 60 日 IC 半衰期 < 20 日 → 标记为衰减因子
3. **相关性去重**: Pearson > 0.7 的因子组保留 IC 最高者

```python
from src.factor.auto_screen import AutoFactorScreen

screener = AutoFactorScreen(
    ic_threshold=0.03,
    icir_threshold=0.3,
    corr_threshold=0.7,
    decay_halflife_min=20,
)
selected_factors = screener.screen(factor_matrix, forward_returns)
```

## 因子衰减监控 (FactorDecayMonitor + CrowdingDetector)

因子有效性会随时间衰减 (alpha decay)。`src/monitoring/factor_monitor.py` 提供因子生命周期的持续监控。

### 核心监控指标

| 指标 | 函数 | 说明 | 阈值 |
|------|------|------|------|
| **滚动 IC** | `rolling_ic()` | 因子值与未来收益截面 Spearman 相关 | `FACTOR_IC_WARNING=0.02` |
| **ICIR** | `calc_icir()` | IC 稳定性 = mean(IC) / std(IC) | `FACTOR_ICIR_THRESHOLD=0.3` |
| **PSI** | `calc_psi()` | 因子分布漂移 (Population Stability Index) | warning=0.1, critical=0.25 |

### FactorDecayMonitor

```python
from src.monitoring.factor_monitor import FactorDecayMonitor

monitor = FactorDecayMonitor()
result = monitor.check_decay(factor_values, forward_returns, dates, reference_values)
# → {"ic_mean": 0.035, "icir": 0.52, "psi": 0.08, "level": "normal", "action": "continue"}
```

分级响应:
- **normal**: IC/ICIR/PSI 均在安全范围 → 继续运行
- **warning**: IC < 0.02 或 ICIR < 0.3 或 PSI > 0.1 → 审查因子、考虑降权
- **critical**: PSI > 0.25 → 立即重训模型

### CrowdingDetector (因子拥挤度)

因子拥挤 = 过多资金追逐同一因子，导致因子溢价消失甚至反转。

| 指标 | 方法 | 阈值 |
|------|------|------|
| **HHI (持仓集中度)** | `herfindahl_index(weights)` | 均匀分布 HHI=1/N，集中度高则拥挤 |
| **成交量异常** | `volume_anomaly(recent, historical)` | ratio > 2.0 → 过热 |
| **因子间相关性** | `cross_factor_correlation(a, b)` | 正常不相关的因子相关性 > 0.5 → 系统性风险 |
| **综合拥挤分** | `detect()` | 0.4×HHI + 0.3×vol + 0.3×corr，> 0.7 → 减仓 |

### 因子 Embedding 去重 (P2-24, 远期)

FactorMiner 发现: LLM 因子挖掘的瓶颈是"反复挖掘同类因子"。解决方案: 维护因子经验记忆库 (已拒绝/已上线因子的 embedding), LLM 生成新因子前先检索去重, 余弦相似度 > 0.85 视为重复。

### 关键警告 (来自 AlphaForgeBench, arXiv:2602.18481)

> **LLM 不应直接执行交易**: LLM 交易智能体行为不稳定。正确做法是让 LLM 产出**可执行 alpha/因子代码**, 然后在**确定性回测引擎**中验证, 通过后才进入策略池。
