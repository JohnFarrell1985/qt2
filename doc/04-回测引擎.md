# 回测引擎 (src/backtest/)

## 概述

支持日线和分钟线两种粒度的策略回测，包含完整的费用计算和绩效统计。

回测的核心目的: **在真金白银下场之前，用历史数据检验策略是否有效**。

## 模块结构

| 文件 | 职责 |
|------|------|
| `engine.py` | 单笔交易计算、多笔汇总 |
| `engine_minute.py` | 分钟线回测引擎 |
| `fees.py` | A股 + 港股通费用计算 |
| `performance.py` | 绩效统计 (夏普/回撤/Calmar等) |
| `data_loader.py` | 从 PostgreSQL 加载行情数据 |
| `strategy_runner.py` | 策略运行器 |
| `stock_picker.py` | 选股器 (`CachedPicker` 读 picks JSON) |
| `cli.py` | 命令行回测入口 |

**单笔交易成本估算** (以 10 万元、万1.15 佣金为例):

## 绩效指标

| 指标 | 公式 | 好的标准 | 说明 |
|------|------|---------|------|
| 年化收益率 | (终值/初值)^(365/天数) - 1 | > 15% | 跑赢沪深300 |
| 最大回撤 | max((peak - trough) / peak) | < 20% | 散户心理承受力 |
| 夏普比率 | (年化收益 - rf) / 年化波动率 | > 1.0 | 风险调整收益 |
| Sortino比率 | (年化收益 - rf) / 下行波动率 | > 1.5 | 只惩罚下跌波动 |
| Calmar比率 | 年化收益 / 最大回撤 | > 1.0 | 收益与回撤的平衡 |
| 胜率 | 盈利交易 / 总交易 | > 50% | 赚钱概率 |
| 盈亏比 | 平均盈利 / 平均亏损 | > 1.5 | 赚多亏少 |

### 指标解读

```
夏普 1.5 + 胜率 55% + 盈亏比 2.0 + 最大回撤 -15%
→ 优秀策略，可以实盘运行

夏普 0.5 + 胜率 45% + 盈亏比 1.2 + 最大回撤 -35%
→ 一般策略，需要优化

夏普 2.5 + 胜率 80% + 盈亏比 5.0 + 最大回撤 -5%
→ 大概率过拟合! 需要检查是否有未来函数
```

## 各策略的回测方法

### Tier 1 规则策略回测

```python
from src.strategy.registry import StrategyRegistry
from src.backtest.strategy_runner import StrategyRunner

registry = StrategyRegistry()

# 选择策略
strat = registry.create("momentum", config={"lookback_days": 20, "top_n": 10})

# 运行回测
runner = StrategyRunner(
    strategy=strat,
    start_date="2023-01-01",
    end_date="2025-12-31",
    initial_capital=500000,
    pool_name="沪深300",  # 标的池
)
result = runner.run()

# 查看结果
print(f"年化收益: {result.annual_return:.2%}")
print(f"最大回撤: {result.max_drawdown:.2%}")
print(f"夏普比率: {result.sharpe:.2f}")
```

### 不同策略推荐的回测区间

| 策略 | 建议回测区间 | 原因 |
|------|------------|------|
| 动量 | 2020~2025 (含牛市+熊市) | 需要覆盖不同市场周期 |
| 反转 | 2022~2025 (含大跌+反弹) | 重点看急跌后的反弹效果 |
| 双低可转债 | 2021~2025 | 可转债品种较新, 数据从 2021 起较完整 |
| 网格交易 | 2023~2025 (震荡阶段) | 震荡市数据越多越好 |
| 低波红利 | 2020~2025 (含极端行情) | 要验证熊市时的防守能力 |
| 多因子打分 | 2020~2025 (越长越好) | 因子稳定性需长期验证 |
| ML 模型 | 测试集 ≠ 训练集 | **绝不能在训练数据上回测!** |

### ML 策略的回测注意事项

```
错误 ✗:  用 2020~2025 数据训练, 再在同样的 2020~2025 回测 → 过拟合!
正确 ✓:  2020~2023 训练, 2024~2025 回测 (时序分割)
```

ML 回测的正确流程:
1. 训练: 2020-01-01 ~ 2023-12-31
2. 验证: 2024-01-01 ~ 2024-12-31 (调参用)
3. 测试: 2025-01-01 ~ 2026-03-31 (最终回测, 只看一次)

### 回测结果对比

建议同时回测多个策略, 对比表现:

| 策略 | 年化收益 | 最大回撤 | 夏普 | 胜率 | 盈亏比 |
|------|---------|---------|------|------|--------|
| 动量 | 25% | -18% | 1.2 | 52% | 1.8 |
| 双低转债 | 12% | -8% | 1.5 | 60% | 1.5 |
| 低波红利 | 10% | -12% | 0.9 | 55% | 1.3 |
| LGB ML | 30% | -15% | 1.8 | 58% | 2.1 |

→ 进攻用 ML + 动量, 防守用转债 + 低波, 由宏观环境切换。

## 分钟线回测

适用于需要在日内精确控制买卖时点的策略 (如网格策略)。

```python
from src.backtest.engine_minute import MinuteBacktestEngine

def my_strategy(bars, position, cash):
    if len(bars) >= 20 and bars[-1]["close"] > bars[-20]["close"]:
        return "buy"
    if position > 0 and bars[-1]["close"] < bars[-5]["close"]:
        return "sell"
    return None

engine = MinuteBacktestEngine(my_strategy, period="5m")
result = engine.run("000001", date(2025, 1, 1), date(2025, 6, 30))
```

### 分钟线回测注意

- 数据量远大于日线, 回测速度较慢
- 建议限制回测标的数量 (单只或少量)
- 滑点影响更大, 建议加入滑点模型

## 回测的常见陷阱

### 1. 未来函数

```
错误: 用 "今天的收盘价" 在 "今天的开盘前" 下单  → 不可能!
正确: 用 "昨天的收盘价" 生成信号, "今天的开盘" 下单
```

本系统信号日期是 trade_date, 实际执行在 T+1 开盘。

### 2. 幸存者偏差

如果只回测现在还在交易的股票, 会遗漏退市的亏损标的, 导致回测收益偏高。
建议: 使用全历史股票列表 (含已退市)。

### 3. 交易成本忽略

```
不含费用的回测:  年化 30%  → "很棒!"
含费用后:         年化 5%   → "不行..."
```

高频换手策略受费用影响很大。本系统所有回测默认包含完整费用。

### 4. 过度优化 (过拟合)

参数在历史数据上调到完美 → 实盘失效。
建议: 使用时序外样本测试, 保持参数简单。

### 5. 多重检验偏差 (Multiple Testing Bias)

10 个策略各跑回测选"最好的"，即使策略全部随机，也有 ~40% 概率出现 Sharpe > 1。

## Deflated Sharpe Ratio (多重检验修正)

`performance.py` 内置了 Bailey & López de Prado (2014) 提出的 Deflated Sharpe Ratio:

```python
from src.backtest.performance import deflated_sharpe_ratio, full_performance_report

# 单独使用
p_value = deflated_sharpe_ratio(
    observed_sharpe=1.5,
    num_trials=20,       # 尝试了多少个策略/参数组合
    var_sharpe=0.04,     # 收益率方差
    skewness=-0.3,       # 收益偏度
    kurtosis=4.0,        # 收益峰度
    T=252,               # 回测天数
)
# p < 0.05 → 策略大概率有真实 alpha

# 集成到完整报告
report = full_performance_report(equity_curve, trades, num_trials=20)
# report["deflated_sharpe_pvalue"] = 0.032
```

| 参数 | 说明 |
|------|------|
| `num_trials` | 尝试的策略/参数数量，越多 DSR 越严格 |
| `expected_max_sharpe()` | 纯随机情况下的期望最大 Sharpe (随 trials 数增长) |
| p-value | < 0.05 → 真实 alpha; > 0.05 → 可能过拟合运气 |

参考: Bailey & López de Prado, *"The Deflated Sharpe Ratio"*, Journal of Portfolio Management 2014
