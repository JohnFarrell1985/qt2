# A股量化因子迭代平台 — 总体设计

## 1. 系统定位

面向 A 股市场的量化投资平台，核心目标是**自动化因子挖掘 + 策略生成 + 实盘交易闭环**。

## 3. 核心流程

### 3.1 数据流

```
                 迅投服务器 (行情中心)
                        │
                  xtquant SDK
                        │
               ┌────────┴────────┐
               │ DownloadEngine  │ ← 分批500只/批间2s暂停/异步等待/断点续传/重试
               └────────┬────────┘
                        │
               MiniQMT 本地缓存
                        │
               get_local_data()
                        │
              行情/财务/因子数据 ──► PostgreSQL
                                       │
                        因子预处理 ◄────┘
                        (去极值/标准化/中性化)
                               │
                        IC/IR 分析
                               │
                        LightGBM 训练
                               │
                        因子组合自动迭代
                               │
                        交易信号生成
                               │
                     ┌─────────┴─────────┐
                     ▼                   ▼
                历史回测            QMT 交易执行
                (日线/分钟线)       (模拟盘/实盘)
```

### 3.2 策略迭代闭环

1. **初始因子池**: 400+ QMT 因子 + 自定义技术因子
2. **IC/IR 筛选**: Spearman Rank IC, 信息比率过滤
3. **LightGBM 训练**: 因子→未来N日收益率回归
4. **自动迭代**: 淘汰低重要性因子，引入新候选，调参数
5. **综合评分**: IC均值×0.3 + ICIR×0.4 + 多空收益×0.3
6. **策略入池**: 达标策略保存至策略池，绑定标的池
7. **宏观映射**: 根据宏观环境状态选择最优策略组合
8. **回测验证**: 夏普/回撤/Calmar 指标验证
9. **实盘执行**: QMT 模拟盘→实盘

## 4. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | Python 3.10+ | 量化生态完善 |
| Web框架 | FastAPI | 异步、自动 OpenAPI 文档 |
| 数据库 | PostgreSQL | 时序数据、复杂查询 |
| ORM | SQLAlchemy 2.0 | 成熟稳定 |
| ML | LightGBM 4.x | 速度快、GPU可选、sklearn兼容 |
| 数据源 | 迅投 xtquant SDK | A股全量数据 + 交易 |
| 配置 | pydantic-settings + .env | 类型安全 + 环境变量 |
| 调度 | schedule + threading | 轻量级 |
| 构建 | uv + hatch | 现代 Python 工具链 |
| 部署 | Docker Compose | 容器化标准 |
| 监控 | OpenClaw + 飞书 | 团队协作 |

## 5. 数据库 ER 概览

### 行情数据
- `stocks` — 股票基础信息
- `stock_daily` — 日线行情
- `stock_minute` — 分钟线行情
- `market_index` — 大盘指数

### 因子数据
- `factor_meta` — 因子元信息
- `factor_values` — 因子值时序数据

### 财务数据
- `stock_financial_report` — 财务报表
- `stock_financial_indicator` — 财务指标

### ML
- `ml_model_log` — 模型训练日志
- `ml_prediction` — 预测结果

### 交易
- `trade_order` — 委托记录
- `trade_position` — 持仓快照
- `trade_daily_report` — 每日绩效

### 策略管理
- `strategy` — 策略定义
- `instrument_pool` — 标的池
- `strategy_allocation` — 策略-标的-宏观关联
- `macro_state_log` — 宏观状态日志

## 6. 安全设计

- QMT 账户信息通过环境变量注入，不入代码/配置文件
- 实盘交易需调用 `confirm_live_mode()` 二次确认
- 风控模块独立于交易逻辑: 止损/止盈/仓位限制/日亏损限额
- 数据库密码通过 `.env` 文件管理，`.gitignore` 排除
