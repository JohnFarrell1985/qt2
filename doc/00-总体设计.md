# AI量化交易迭代平台(A股散户版) — 总体设计 (v5.0)

## 1. 系统定位

面向 **A 股散户 T+1 低频交易** 的量化投资平台，核心目标:

1. **自动化因子挖掘**: 400+ QMT 因子 + Alpha158 量价因子 → IC 筛选 + alphalens 质量门控 → LightGBM 自动迭代 → 最优因子组合; 远期引入 LLM 进化式因子挖掘 (QuantaAlpha 风格)
2. **三档策略引擎 + ETF 轮动**: 规则策略 (简单可靠) + 多因子打分 (系统化) + ML 模型 (高收益潜力) + ETF 全球资产配置 (VAA/DAA/CAA)
3. **Regime 感知买卖体系**: 策略信号 → Regime 门控过滤 → 持仓监控 → 信号仲裁 → 分数 Kelly + 回撤自适应仓位分配
4. **实盘交易闭环**: 策略信号 → 操作清单 → QMT 下单 (模拟盘 / 实盘)
5. **研究驱动增强**: 北向资金/跨资产 Regime 信号 → 情绪引擎 → 宏观状态分类; RAG 投研知识库 (远期)

## 3. 核心流程

### 3.1 数据流

```
                 迅投服务器 (行情中心)
                        │
                  xtquant SDK
                        │
               ┌────────┴────────┐
               │ DownloadEngine  │ ← 分批500只/批间2s暂停/异步等待/断点续传/重试
               └────────┬────────┘
                        │
               QMT 本地缓存
                        │
               get_local_data()
                        │
              行情/财务/因子数据 ──► PostgreSQL
                                       │
                        因子预处理 ◄────┘
                        (去极值/标准化/中性化)
                               │
                        IC/IR 分析
                               │
                        LightGBM 训练
                               │
                        因子组合自动迭代
                               │
                        交易信号生成
                               │
                     ┌─────────┴─────────┐
                     ▼                   ▼
                历史回测            QMT 交易执行
                (日线/分钟线)       (模拟盘/实盘)
```

### 3.2 策略迭代闭环

1. **初始因子池**: Alpha158 量价因子 (158 个) + 迅投基本面因子 (~100 个) + QMT 原生因子 (400+) + 自定义技术因子
2. **因子质量门控**: alphalens 标准分析 (IC/分层/换手) + 自动筛选 (IC>0.03, ICIR>0.3, 衰减检测, 相关性去重)
3. **LightGBM 训练**: 因子→未来N日收益率回归 (Purged Walk-Forward CV)
4. **自动迭代**: 淘汰低重要性因子，引入新候选，调参数; Bandit 自动选择"挖因子"或"调模型" (RD-Agent 风格)
5. **综合评分**: IC均值×0.3 + ICIR×0.4 + 多空收益×0.3 + Deflated Sharpe Ratio 多重检验修正
6. **Regime 门控**: 根据市场 drift/volatility 状态, 仅放行适合当前 regime 的策略信号
7. **策略入池**: 达标策略保存至策略池，绑定标的池
8. **宏观映射**: 情绪合成指数 + 北向资金流 + 跨资产 regime → 宏观状态 → 策略参数 Profile
9. **组合优化**: CAA (Keller 2015) / Riskfolio-Lib 风险预算 / HRP → 分数 Kelly + 回撤自适应
10. **回测验证**: 夏普/回撤/Calmar + DSR 过拟合检验
11. **实盘执行**: QMT 模拟盘→实盘

### 3.3 每日操作流 (v5.0)

```
09:15  查询 QMT 持仓 + 资金
         │
         ▼
       POST /strategy/execute
         │
         ├─ 策略信号生成 (10个策略 × 买+卖)
         ├─ PositionMonitor 扫描 (止损/止盈)
         ├─ SignalArbiter 仲裁 (去重/T+1/投票)
         └─ PositionSizer 分配 (等权/ATR)
         │
         ▼
       操作清单 [{sell: A, B}, {buy: C, D}]
         │
09:30    执行卖出 ──► QMT
10:00    确认成交 → 资金释放
10:30    执行买入 ──► QMT
15:00    收盘
15:30    持仓快照 + 日报
17:00    增量数据同步
```

## 4. 三档策略引擎

| 档位 | 类型 | 策略 (10个) | 适用场景 | 训练方式 |
|------|------|------------|---------|---------|
| Tier 1 | 规则 | 动量 | 牛市趋势追随 | 无需训练, 调参即用 |
| Tier 1 | 规则 | 反转 | 底部复苏抄底 | 无需训练 |
| Tier 1 | 规则 | 行业轮动 | 结构性行情 | 无需训练 |
| Tier 1 | 规则 | 均线突破 | 温和趋势 | 无需训练 |
| Tier 1 | 规则 | T+1 宽网格 | 震荡区间 | 无需训练 |
| Tier 1 | 规则 | 双低可转债 | 熊市/震荡防守 | 无需训练 |
| Tier 1 | 规则 | 低波红利 | 熊市防守 | 无需训练 |
| Tier 2 | 打分 | 多因子等权/IC加权 | 系统化选股 | IC 检验选因子 |
| Tier 3 | ML | LightGBM | 非线性因子组合 | 自动迭代训练 |
| Tier 3 | ML | XGBoost*/CatBoost* | 备选模型 | 同上 (预留) |

详见 [04-策略管理](04-策略管理.md)。

## 5. 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | Python 3.10+ | 量化生态完善 |
| Web框架 | FastAPI | 异步、自动 OpenAPI 文档 |
| 数据库 | PostgreSQL | 时序数据、复杂查询 |
| ORM | SQLAlchemy 2.0 | 成熟稳定 |
| ML | LightGBM 4.x | 速度快、GPU可选、sklearn兼容 |
| 数据源 | 迅投 xtquant SDK | A股全量数据 + 交易 |
| 配置 | pydantic-settings + .env | 类型安全 + 环境变量 |
| 调度 | schedule + threading | 轻量级 |
| 构建 | uv + hatch | 现代 Python 工具链 |
| 部署 | Docker Compose | 容器化标准 |

## 6. 数据库 ER 概览

### 行情数据
- `stocks` — 股票基础信息
- `stock_daily` — 日线行情
- `stock_minute` — 分钟线行情
- `market_index` — 大盘指数

### 因子数据
- `factor_meta` — 因子元信息
- `factor_values` — 因子值时序数据

### 财务数据
- `stock_financial_report` — 财务报表
- `stock_financial_indicator` — 财务指标

### ML
- `ml_model_log` — 模型训练日志
- `ml_prediction` — 预测结果

### 交易
- `trade_order` — 委托记录
- `trade_position` — 持仓快照
- `trade_daily_report` — 每日绩效

### 策略管理
- `strategy` — 策略定义
- `instrument_pool` — 标的池 (支持多资产类型: A股主板/科创/创业/北交/港股通/ETF/可转债/两融)
- `strategy_allocation` — 策略-标的-宏观关联
- `macro_state_log` — 宏观状态日志

### 标的池资产类型

不同资产类型有独立的交易规则 (T+N、涨跌幅、印花税、最小交易单位), 由 `TradingRule` 引擎统一管理:
- `a_stock_main` — A 股主板 (T+1, ±10%)
- `a_stock_star` — 科创板 (T+1, ±20%, 200 股起)
- `a_stock_gem` — 创业板 (T+1, ±20%)
- `a_stock_bse` — 北交所 (T+1, ±30%)
- `hk_connect` — 港股通 (T+0, 无涨跌幅)
- `etf_domestic` — 境内 ETF (T+1)
- `etf_cross_border` — 跨境/商品 ETF (T+0)
- `cb` — 可转债 (T+0, ±20%)
- `margin_long/short` — 两融标的

## 7. 安全设计

- QMT 账户信息通过环境变量注入，不入代码/配置文件
- 实盘交易需调用 `confirm_live_mode()` 二次确认
- 风控模块独立于交易逻辑: 止损/止盈/仓位限制/日亏损限额
- 数据库密码通过 `.env` 文件管理，`.gitignore` 排除
- 多层风控: 策略级 → 持仓级 → 组合级 → 资金级 → 执行级
