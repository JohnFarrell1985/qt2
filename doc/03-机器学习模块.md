# 机器学习模块 (src/ml/)

## 概述

使用 LightGBM (可扩展 XGBoost / CatBoost) 对因子组合建模，预测个股未来收益率，自动迭代优化因子组合。

**核心价值**: 人工选因子打分是 Tier 2 (线性加权)，ML 策略是 Tier 3——它能自动发现因子之间的非线性关系和交互效应，比人工打分更强。

## 模块结构

| 文件 | 职责 |
|------|------|
| `dataset.py` | 构建训练数据集 (因子矩阵 + 标签) |
| `lgb_model.py` | LightGBM 模型封装 (训练/预测/保存) |
| `feature_selection.py` | IC/相关性因子筛选 |
| `model_evaluation.py` | 模型评估 (IC/ICIR/分组收益) |
| `strategy_builder.py` | 预测结果→交易信号 |
| `auto_iterate.py` | 自动迭代优化引擎 |

## 适用场景

### 什么时候该用 ML 策略

| 场景 | 是否适合 | 原因 |
|------|---------|------|
| 因子数量 > 20 个 | 非常适合 | 人工加权效率低，ML 自动寻优 |
| 因子关系非线性 | 非常适合 | ML 捕捉交互效应 (如低 PE + 高动量) |
| 有 1 年以上日频数据 | 适合 | 训练样本足够 |
| 因子只有 3~5 个 | 不推荐 | 数据量不够发挥 ML 优势，用 Tier 2 打分即可 |
| 日内高频策略 | 不推荐 | 本系统面向 T+1 日频 |
| 无历史数据 | 不可用 | ML 必须有历史训练集 |

### 与 Tier 1 / Tier 2 的区别

```
Tier 1 (规则):  if 涨幅 > 5% then 买    → 简单、可解释、但维度单一
Tier 2 (打分):  score = 0.3×ROE + 0.4×动量 + 0.3×低波  → 多维度、但只有线性组合
Tier 3 (ML):    model.predict(30个因子) → 0.032  → 非线性、自动、但黑箱
```

**建议**: 三档同时运行，由 SignalArbiter 投票选择最终操作。当 ML 和规则策略同时看多某只股票时，信心更强。

## LightGBM 模型

### 为什么选 LightGBM

| 特性 | LightGBM | XGBoost | CatBoost |
|------|----------|---------|----------|
| 训练速度 | 最快 | 中等 | 较慢 |
| 内存占用 | 最小 | 中等 | 较大 |
| 类别特征 | 原生支持 | 需编码 | 原生最优 |
| 过拟合控制 | leaf-wise (需调参) | level-wise (更稳) | 有 ordered boost |
| A 股适用性 | 首选 | 备选 | 备选 |

**结论**: LightGBM 速度快、内存小，适合 A 股 5000+ 标的的日频预测。XGBoost/CatBoost 作为备选，通过 `_create_model()` 接口一键切换。

### 默认参数

```python
{
    "objective": "regression",        # 回归任务 (预测收益率)
    "metric": "mse",                  # 均方误差
    "boosting_type": "gbdt",          # 标准梯度提升
    "num_leaves": 63,                 # 叶子数 (控制复杂度)
    "learning_rate": 0.05,            # 学习率
    "n_estimators": 1000,             # 最大迭代轮数
    "subsample": 0.8,                 # 行采样比例
    "colsample_bytree": 0.8,          # 列采样比例
    "importance_type": "gain",        # 特征重要性用增益
}
```

### 参数调优建议

| 参数 | 过拟合时 | 欠拟合时 | 说明 |
|------|---------|---------|------|
| `num_leaves` | 减小 (31) | 增大 (127) | 越大模型越复杂 |
| `learning_rate` | 减小 (0.01) | 增大 (0.1) | 越小需要更多轮 |
| `n_estimators` | 不需改 (由 early_stopping 决定) | 增大 (2000) | 配合低 lr 使用 |
| `subsample` | 减小 (0.6) | 增大 (1.0) | 随机采样防过拟合 |
| `min_child_samples` | 增大 (50) | 减小 (10) | 叶子最少样本数 |
| `reg_alpha` | 增大 (0.1) | 减小 (0) | L1 正则化 |
| `reg_lambda` | 增大 (1.0) | 减小 (0) | L2 正则化 |

### Early Stopping

LightGBM >= 4.0 要求通过 callback 传递：

```python
callbacks=[
    lgb.early_stopping(stopping_rounds=50),
    lgb.log_evaluation(period=100),
]
```

当验证集上连续 50 轮无改善时自动停止，防止过拟合。

### 模型序列化

使用 `joblib.dump/load` 保存完整 sklearn 包装器：

```python
import joblib
joblib.dump(model, "models/best_lgb.pkl")
model = joblib.load("models/best_lgb.pkl")
```

保留 `best_iteration_` 等属性，确保预测使用最佳轮数。

## 数据集构建

```
因子数据 (factor_values 表)
    │
    ▼  get_factor_values() → MultiIndex(trade_date, code) DataFrame
    │
    ▼  _calc_forward_returns() → 未来N日收益率标签
    │      └─ ML_LABEL_PERIOD=2 (散户: 隔日收益，匹配 T+1)
    │
    ▼  内连接 → (X: 因子矩阵, y: 标签)
    │
    ▼  时序分割 (60%训练 / 20%验证 / 20%测试)
```

### 标签设计 (ML_LABEL_PERIOD)

| 值 | 含义 | 适用 |
|----|------|------|
| 1 | 次日收益率 | 超短线 (T+0 场景, A 股不推荐) |
| 2 | 隔日收益率 | **A 股 T+1 散户推荐** (今天信号, 明天买, 后天卖) |
| 5 | 5 日收益率 | 周度调仓 |
| 20 | 月度收益率 | 月度调仓 (稳定但迟钝) |

**`.env` 设置**: `ML_LABEL_PERIOD=2`

### 时序分割 (重要!)

```
2016          2020          2023          2025    2026
 |————训练集 60%————|——验证集 20%——|——测试集 20%——|
                                                  ↑ 实盘

切勿用随机分割! 金融数据有时序依赖, 必须按时间切割。
```

## 因子筛选 (feature_selection.py)

### 为什么要筛选因子

- 400+ QMT 因子全用 → 维度灾难 + 严重过拟合
- 有效因子通常只有 20~50 个

### 筛选流程

```
400+ 候选因子
    │
    ▼  Step 1: IC 检验
    │    IC均值 > 0.02  → 通过 (约 30%)
    │    ICIR > 0.3     → 加分
    │
    ▼  Step 2: 相关性去重
    │    因子间 |corr| > 0.7 → 保留 IC 更高的那个
    │    避免共线性
    │
    ▼  Step 3: 初始因子集 (约 30 个)
    │
    ▼  Step 4: LightGBM 训练
    │    feature_importances_ (gain)
    │    底部 20% 因子淘汰
    │
    ▼  Step 5: 自动迭代 (AutoIterateEngine)
    │    引入新因子 → 重训 → 评估 → 循环
    │
    ▼  最终因子集 (15~30 个)
```

## 自动迭代优化 (AutoIterateEngine)

### 工作流程

```
1. IC/IR 初筛 → top_n 因子
        │
2. LightGBM 训练 + 验证
        │
3. 综合评分 = IC×0.3 + ICIR×0.4 + 多空收益×0.3
        │
4. 因子进化:
   - 淘汰底部 20% 低重要性因子
   - 随机引入新候选因子
   - 每3轮探索3个新因子
   - 每5轮随机调参 (num_leaves, lr, n_estimators)
        │
5. 收敛判定:
   - 达到 target_sharpe → 停止
   - 连续 patience 轮无改善 → 停止
   - 达到 max_iterations → 停止
        │
6. 输出: 最佳因子组合 + 权重 + 模型
```

### 迭代参数 (`.env`)

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ML_MAX_ITERATIONS` | 50 | 最大迭代轮数 |
| `ML_TARGET_SHARPE` | 1.5 | 目标夏普比率 (达标即停) |
| `ML_PATIENCE` | 10 | 连续无改善容忍轮数 |
| `ML_LABEL_PERIOD` | 2 | 标签周期 (隔日收益) |

### 统计分析

- **因子频率**: 统计各因子在所有迭代中出现次数 → 高频因子是稳定有效因子
- **收敛曲线**: 每轮得分 + 累计最佳得分 → 判断是否还需继续迭代
- **因子重要性**: LightGBM `feature_importances_` (gain) → 排序因子贡献
- **DB 记录**: 每轮结果存 `ml_model_log` 表，可追溯

## 模型重训计划

### 推荐频率

| 市场状态 | 重训频率 | 原因 |
|---------|---------|------|
| 稳定期 | 每 4 周 | 因子有效性缓慢衰减 |
| 风格切换 | 每 2 周 | 因子表现可能反转 |
| 重大事件 | 立即重训 | 政策变化导致因子失效 |

### 重训流程

```
1. 扩展训练集 (加入最新数据)
        │
2. 重新训练 (自动迭代 5~10 轮微调)
        │
3. 对比新旧模型在测试集上的表现
        │
4. 新模型更优 → 替换; 否则 → 保留旧模型
        │
5. 更新 STRAT_ML_MODEL_PATH 或直接覆盖 ./models/best_lgb.pkl
```

定时任务已内置: 每 4 周自动重训 (见 `scheduler`)。

## 常见问题

### Q: 模型预测值是什么含义?
预测的是 **未来 N 日的截面收益率** (N = `ML_LABEL_PERIOD`)。预测值 0.03 意味着模型认为这只股票未来 2 日会比平均水平高 3%。

### Q: 预测值和买入阈值怎么设?
`STRAT_ML_LONG_THRESHOLD=0.0` 意味着只要预测收益 > 0 就有买入候选资格。实际中可以设高一些 (如 0.01) 减少噪音。

### Q: 为什么模型回测很好但实盘不行?
常见原因: (1) 过拟合——加大正则化、减少因子数; (2) 未来函数——检查财务数据是否按公告日取; (3) 市场风格切换——定期重训。

### Q: 如何切换到 XGBoost / CatBoost?
修改 ML 策略的 `_create_model()` 方法，返回对应的模型实例。接口已预留 `xgboost_ml` 和 `catboost_ml` 注册名。
