# 市场情绪引擎 (src/sentiment/)

## 概述

市场情绪引擎是整个量化平台的**大脑中枢**——它负责从多个数据源收集市场情报，计算综合情绪指数，自动判断宏观环境状态，并为每个策略选择对应的参数配置。

**核心目标**: 将当前系统中纯手动的宏观状态判断，升级为"数据驱动建议 + 人工确认"的半自动模式。

```
  ┌─ src/datacollect/ ─────────────────────────────┐
  │  多源情报采集 (5 层数据源, 反爬, 限流)          │
  └────────────────────┬───────────────────────────┘
                       │
  ┌─ src/dataclean/ ───┴───────────────────────────┐
  │  instructor + LLM 清洗 (DeepSeek/Qwen → JSON)   │
  │  Pydantic 自动校验 + 指数退避重试 + 三级降级     │
  └────────────────────┬───────────────────────────┘
                       │
  ┌─ src/sentiment/ ───┴───────────────────────────┐
  │  情绪特征工程 (滚动/Z-score/差分/合成)          │
  │       │                                        │
  │  合成情绪指数 (-1 恐慌 ~ +1 贪婪)              │
  │       │                                        │
  │  宏观状态分类器 (规则引擎)                      │
  │       │  建议: "当前应切换到震荡市"              │
  │       │                                        │
  │  策略参数 Profile 切换                          │
  └────────────────────┬───────────────────────────┘
                       │
  StrategyOrchestrator (动态参数执行)
```

**三层独立模块**: 采集 (`datacollect`) → 清洗 (`dataclean`) → 分析 (`sentiment`)

> 相关文档: [12-数据采集模块](12-数据采集模块.md) | [13-数据清洗与LLM](13-数据清洗与LLM.md)

## 设计原则

1. **不依赖单一数据源** — 任何一个源挂了都不影响整体判断
2. **零成本优先** — 从已有数据库计算的量价情绪是核心骨架
3. **避免限流** — 每日仅 3-4 次定时采集，总请求 < 25 次/天
4. **可扩展** — 用户可通过数据源注册表自由添加/禁用数据源
5. **半自动** — 系统建议宏观状态，人可确认或覆盖
6. **LLM 清洗优先** — 非结构化数据由大模型清洗为标准 JSON 后再入库

## 五层数据源

> 各层的采集实现见 [12-数据采集模块](12-数据采集模块.md)。

### Layer 1: 量价情绪 (零成本, 最高优先级)

从已有 PostgreSQL `stock_daily` 表截面计算，**不产生任何网络请求**。

| 指标 | 计算方式 | 含义 |
|------|---------|------|
| 涨跌比 (A/D Ratio) | 上涨家数 / 下跌家数 | >1.5 乐观, <0.5 恐慌 |
| 涨停家数 | 涨幅 >= 9.8% 的股票数 | 市场赚钱效应 |
| 跌停家数 | 跌幅 <= -9.8% 的股票数 | 市场亏钱效应 |
| 炸板率 | 曾涨停后打开的 / 总涨停 | 封板质量 |
| 创 60 日新高家数 | close = max(close, 60d) | 趋势强度 |
| 创 60 日新低家数 | close = min(close, 60d) | 恐慌深度 |
| 市场波动率 (5/10/20 日) | 全市场日收益率标准差 | 波动率飙升=恐慌 |
| 缩放量强度 | 全市场成交额 / 20日均成交额 | >1.3 放量, <0.7 缩量 |
| 板块集中度 | 涨幅 Top3 板块涨停股 / 全市场涨停 | 高=抱团, 低=普涨 |

### Layer 2-5 概览

| 层级 | 数据来源 | 频率 | 主要指标 |
|------|---------|------|---------|
| Layer 2 | xtdata (QMT 本地缓存) | 按需 | 板块热度, 指数位置, ETF 溢折价 |
| Layer 3a | akshare 直调 (免费) | <15次/天 | 北向资金, 融资, 涨停池, 汇率, 期货基差 |
| Layer 3b | OpenClaw 推送 | 4次/天 | 新闻汇总, A股分析, 黄金/大宗 |
| Layer 4 | Tavily 搜索 | 按需 | 补盲/兜底 (免费 1000次/月) |
| Layer 5 | Browser 自动化 | 1-2次/天 | 雪球热帖, 同花顺板块 |

## 情绪特征工程

### 原始指标 → 模型特征

对每个原始情绪指标，生成 5 类衍生特征:

```
原始指标 (如 ad_ratio = 1.3)
    │
    ├── 滚动均值: ad_ratio_ma5, ad_ratio_ma10, ad_ratio_ma20
    ├── 偏离度:   ad_ratio_dev = (当前 - MA20) / MA20
    ├── 差分:     ad_ratio_diff = 今日 - 昨日 (变化速度)
    ├── Z-score:  ad_ratio_zscore = (当前 - 60日均) / 60日标准差
    └── 分位数:   ad_ratio_pctile = 在过去 250 日中的百分位
```

### 情绪分类

| 维度 | 指标组合 | 输出 |
|------|---------|------|
| 赚钱效应 | 涨跌比 + 涨停数 + 创新高数 | earning_effect: -1 ~ +1 |
| 资金情绪 | 北向净流入 + 融资余额变化 | capital_mood: -1 ~ +1 |
| 波动情绪 | 市场波动率 + 跌停数 + 缩放量 | volatility_mood: -1 ~ +1 |
| 板块热度 | 板块集中度 + 热门板块数 | sector_heat: -1 ~ +1 |
| 新闻情绪 | OpenClaw 新闻评分 + 事件影响 | news_mood: -1 ~ +1 |
| 外围情绪 | 汇率变动 + 黄金 + 原油 | global_mood: -1 ~ +1 |

### 合成情绪指数 (Composite Sentiment Index)

将 **8 个维度** 加权合成为单一指数 (v2.0, 研究驱动增强):

```
CSI = w1 × earning_effect     (权重 0.20, 最直接)
    + w2 × capital_mood        (权重 0.20, 最准确)
    + w3 × volatility_mood     (权重 0.12, 前瞻)
    + w4 × sector_heat         (权重 0.08, 结构)
    + w5 × news_mood           (权重 0.12, 事件驱动)
    + w6 × global_mood         (权重 0.08, 外围)
    + w7 × northbound_flow     (权重 0.10, 聪明钱, P1-33 新增)
    + w8 × cross_asset_regime  (权重 0.10, 跨资产, P1-37 新增)
```

输出范围: **-1.0 (极度恐慌) ~ +1.0 (极度贪婪)**

权重可通过 `.env` 配置调整。

#### 新增维度: 北向资金流信号 (P1-33, 研究驱动)

北向资金是 A 股最公开的"聪明钱"代理 (Pacific-Basin Finance Journal 2025 实证):
- **5 日净流入 Z-score**: 短期资金动向
- **20 日净流入趋势**: 中期资金方向
- **Regime 判断**: Z > 0.5 = risk_on, Z < -0.5 = risk_off
- 数据源: AKShare `stock_hsgt_north_net_flow_in` (免费, 每日更新)

#### 新增维度: 跨资产 Regime 上下文 (P1-37, 研究驱动)

跨资产信号领先于 A 股 (散户反应滞后), 监控以下资产:
- **黄金 ETF (518880.SH)**: 避险情绪
- **美元/人民币**: 资本流动方向
- **美国 10 年国债**: 全球利率环境
- **VIX**: 全球恐慌指标
- **铜期货**: 经济先行指标
- 综合判断: 各资产 20 日动量的 risk_on 占比 → cross_asset_risk_score (0~1)

## 策略参数 Profile

### 核心概念

同一策略在不同宏观环境下使用不同参数。例如动量策略:

| 宏观状态 | lookback | top_n | stop_loss | take_profit | max_hold | 说明 |
|---------|----------|-------|-----------|-------------|----------|------|
| 强势牛市 | 10 | 15 | -10% | 25% | 8天 | 激进: 追强势, 宽止损, 高止盈 |
| 温和牛市 | 20 | 10 | -8% | 15% | 5天 | 默认: .env 中的标准参数 |
| 震荡市 | 30 | 5 | -5% | 10% | 3天 | 保守: 选强中强, 快进快出 |
| 温和熊市 | 5 | 3 | -3% | 5% | 2天 | 极保守: 仅做超短线 |
| 深度熊市 | — | — | — | — | — | 禁用: 不启用动量策略 |
| 底部复苏 | 15 | 8 | -6% | 12% | 5天 | 适中: 关注超跌反弹 |

### Profile 加载逻辑

```python
def get_strategy_config(strategy_name: str, macro_state: str) -> dict:
    profiles = load_profiles("strategy_profiles.json")
    
    if strategy_name not in profiles:
        return {}
    
    state_config = profiles[strategy_name].get(macro_state)
    
    if state_config is None:
        return {}
    
    return state_config

# orchestrator.py
profile = get_strategy_config(alloc.strategy_name, current_state)
base_config = _default_config_from_settings()
merged_config = {**base_config, **profile}
strat = registry.create(alloc.strategy_name, config=merged_config)
```

## 数据库表设计 (PostgreSQL JSONB)

### 设计理念: 结构化 + JSONB 混合

```
sentiment_daily 表:
┌──────────────────────────────────────────────────────────┐
│  固定字段 (FLOAT/INT/VARCHAR)       JSONB 字段            │
│  ═══════════════════════════       ═════════             │
│  ad_ratio         1.72             hot_sectors           │
│  north_net_flow   86.0             ["银行","券商","地产"]  │
│  composite_index  0.52             key_events            │
│  macro_state      "bull_moderate"  [{event, impact}...]  │
│  ...                               hot_stocks            │
│                                    [{code, reason}...]   │
│                                    extra                 │
│                                    {gold, oil, fx...}    │
│                                                          │
│  ←── 高频计算/排序/JOIN/LGB ──►    ←── 格式多变/扩展 ──►   │
└──────────────────────────────────────────────────────────┘
```

| 字段类型 | 用 FLOAT/INT/VARCHAR | 用 JSONB |
|---------|---------------------|----------|
| 适用场景 | 格式固定、需要聚合/排序/JOIN | 格式多变、嵌套结构、扩展性 |
| 查询速度 | 最快 (B-tree 索引) | 较快 (GIN 索引) |
| 适合什么 | ad_ratio, north_flow, composite_index | key_events, hot_sectors, hot_stocks |
| 是否用于 LGB | ✅ 直接取值 | ❌ 需要先提取为数值 |

### JSONB 操作符速查表

| 操作符 | 含义 | 示例 |
|--------|------|------|
| `->` | 取 JSON 对象的键 (返回 JSONB) | `extra->'market_mood_text'` |
| `->>` | 取 JSON 对象的键 (返回 TEXT) | `extra->>'market_mood_text'` |
| `@>` | 左边包含右边 | `hot_sectors @> '"银行"'::JSONB` |
| `?` | 对象中是否存在某个键 | `extra ? 'etf_premium_300'` |
| `?|` | 数组中存在任一元素 | `hot_sectors ?| ARRAY['银行','券商']` |
| `?&` | 数组中全部存在 | `hot_sectors ?& ARRAY['银行','券商']` |
| `||` | JSONB 合并 | `extra || '{"new_key":"val"}'` |
| `#>` | 按路径取 JSONB | `key_events#>'{0,event}'` |
| `#>>` | 按路径取 TEXT | `key_events#>>'{0,event}'` |

> 对 JSONB 列创建 GIN 索引后, `@>`, `?`, `?|`, `?&` 都会走索引。

async def run_collect(schedule_slot: str):
    logger.info(f"开始情绪采集: {schedule_slot}")
    
    sources = registry.get_sources_for_slot(schedule_slot)
    
    for source in sources:
        try:
            if source.type == "local_db":
                raw = await collect_price_volume(source)
            elif source.type == "akshare":
                raw = await collect_akshare(source)
            elif source.type == "openclaw_push":
                continue
            elif source.type == "tavily_search":
                raw = await collect_tavily(source)
            else:
                continue
            
            cleaned = await cleaner.process(source.name, raw)
            await save_to_db(source, raw, cleaned)
            
        except Exception as e:
            await log_failure(source, str(e))
            logger.warning(f"采集失败 [{source.name}]: {e}")


async def run_compute_and_classify():
    today = date.today()

        await notify_state_change(result)

## 情绪因子效果评估

### 为什么需要评估

A 股实证研究表明新闻情绪因子 IC 通常在 0.02-0.05 之间，远低于资金因子和量价因子。情绪因子在市场低迷时更有效 (月超额 ~1.62%)，在乐观期效果大幅衰减 (~0.31%)。不做评估 = 无法判断情绪因子是否真正贡献了 alpha，还是只在增加噪声。

### 评估指标体系

| 指标 | 计算方式 | 有效阈值 | 监控频率 |
|------|---------|---------|---------|
| 滚动 IC | `corr(news_mood_t, forward_return_t+1)` 20 日滚动 | \|IC\| > 0.02 | 每日 |
| 滚动 ICIR | `mean(IC_20d) / std(IC_20d)` | ICIR > 0.3 | 每日 |
| IC 正比例 | 近 60 日 IC > 0 的比例 | > 55% | 每周 |
| 增量贡献 | LGB 含/不含情绪因子的 Sharpe 差 | ΔSharpe > 0.05 | 每月回测 |

### 自动降权机制

```python
def evaluate_news_factor(ic_series: pd.Series, window: int = 30) -> dict:
    """评估新闻情绪因子有效性, 动态调整权重"""
    recent_ic = ic_series.tail(window)
    ic_mean = recent_ic.mean()
    ic_positive_ratio = (recent_ic > 0).mean()

    if abs(ic_mean) < 0.01 and ic_positive_ratio < 0.50:
        action = "disable"       # 连续 30 天 IC < 0.01 → 降权至 0
        recommended_weight = 0.0
    elif abs(ic_mean) < 0.02:
        action = "reduce"        # IC 偏低 → 降至默认权重的 50%
        recommended_weight = 0.075  # 默认 0.15 × 50%
    else:
        action = "normal"
        recommended_weight = 0.15   # 默认权重

    return {
        "ic_mean": ic_mean,
        "ic_positive_ratio": ic_positive_ratio,
        "action": action,
        "recommended_weight": recommended_weight,
    }
```

### 消融实验 (Ablation Study)

每月进行一次对比回测，验证情绪因子的实际增量贡献:

| 因子集 | 对比内容 |
|--------|---------|
| A: 量价 + 基本面 (无情绪) | 基线 |
| B: A + 量价情绪 (Layer 1) | 评估零成本量价情绪的贡献 |
| C: B + 资金情绪 (北向/融资) | 评估资金面情绪的贡献 |
| D: C + 新闻情绪 (LLM/蒸馏) | 评估 LLM 新闻情绪的增量 |

若 D 的 Sharpe 与 C 无显著差异 (t-test p > 0.05), 则新闻情绪因子的边际贡献不足, 应降权或暂停。

### 情绪因子贡献排序 (A 股实证)

```
资金情绪特征 (北向/融资) > 量价情绪特征 (涨跌比/波动率)
    > 衍生品特征 (期货基差) > 新闻情绪特征 (NLP 评分)
```

**含义**: Layer 1 和 Layer 3a (akshare 资金数据) 是核心, 新闻情绪是锦上添花。即使 LLM 清洗完全不可用, 系统仍有 70%+ 的情绪分析能力。

## 热门股 / 一日游识别 (进阶)

| 指标 | 计算方式 | 说明 |
|------|---------|------|
| RPS 相对强度 | 个股涨幅排名 / 全市场总数 × 100 | ≥85 = 强势股 |
| 涨停频率 | 近 3 日涨停次数 | ≥2 = 连板 |
| 换手率热度 | 换手率 / 20 日均换手率 | ≥2x = 过热 |
| 封板质量 | 封成比 × (1 - 炸板率) | <0.5 = 一日游风险 |
| 板块联动 | 所属板块涨停家数 | <3 = 孤掌难鸣 |

一日游预警:
- 量化席位主导 + 无一字板 + 次日回撤 > 1.5% → 一日游概率 > 80%
- 可训练 LGB 二分类模型 (目标: 次日跌幅 ≥ 2% = 一日游)

## 常见问题

### Q: 情绪引擎的核心价值是什么?
将"拍脑袋切换宏观状态"升级为"数据驱动建议 + 人工确认"。例如系统自动检测到北向资金持续流入+涨跌比走高+新闻偏乐观，建议切换到牛市参数，你只需确认即可。

### Q: 策略参数 Profile 和 .env 参数冲突怎么办?
Profile 覆盖 .env。加载顺序: `.env` 默认值 → `strategy_profiles.json` 覆盖。如果 Profile 值为 `null`，使用 `.env` 全部默认参数。

### Q: 合成情绪指数的权重怎么调?
通过 `.env` 中 `SENTIMENT_W_*` 参数调整。也可以用 LGB 自动学习最优权重 (P2 阶段)。

### Q: 不用 OpenClaw 能运行吗?
能。Layer 1 (量价) + Layer 3a (akshare) 就能覆盖核心功能。OpenClaw 是增强项不是必需项。

### Q: akshare 被限流了怎么办?
系统有 5 层数据源。Layer 1 完全不走网络。akshare 某接口失败会自动 fallback 到 Tavily 搜索或 Browser 提取。每日总请求量 < 25 次，正常不触发限流。

### Q: JSONB 查询性能够吗?
完全足够。每天 1 行 `sentiment_daily`，十年 2500 行。加上 GIN 索引，任何查询都在毫秒级。

### Q: 定时任务不执行怎么排查?

```python
curl http://localhost:8012/api/sentiment/scheduler-status

SELECT source_name, schedule_slot, status, error_message, collected_at
FROM sentiment_ingest_log
WHERE trade_date = CURRENT_DATE
ORDER BY collected_at DESC;
```

### Q: 家用硬件能跑本地 NLP 吗?
FinBERT2-Base (125M) ONNX INT8 量化后 ~60MB, 纯 CPU 推理 ~5ms; Qwen3-0.6B GGUF Q4 ~400MB, CPU 推理 ~50ms。两模型合计 < 500MB, 任何现代笔记本均可部署。详见 [13-数据清洗与LLM](13-数据清洗与LLM.md) 模型选型章节。

### Q: 新闻情绪应该用什么 LLM?

**三层模型分工 (机构最佳实践)**:

| 层级 | 模型 | 用途 | 成本 |
|------|------|------|------|
| 生产推理: 分类 (目标态) | FinBERT2-Base 125M 本地 ONNX | 情绪分类 (-1/0/+1), 95% 流量 | 0 元 |
| 生产推理: 抽取 (目标态) | Qwen3-0.6B + LoRA 本地 GGUF | 结构化 JSON 抽取 | 0 元 |
| 复杂任务 + 蒸馏标注 | DeepSeek V3 API | 事件抽取、研报分析、蒸馏标注 | ~1 元/百万 token |
| 备选教师 + 降级 | **Qwen3-Max** (百炼旗舰) | DeepSeek 不可用时的备选 + 蒸馏教师 | ¥8.8+¥44/百万 token |

**为什么不用百炼作为主力?**
- DeepSeek V3 日常主力 (¥2/M token 输出); Qwen3-Max 备选+教师 (¥44/M token, 调用极少质量最高)
- 情绪分类是典型的 encoder 任务, 用 decoder 模型 (Qwen-Max) 是"杀鸡用牛刀"
- 头部机构 (九坤/幻方/Two Sigma) 均不依赖云端 LLM API, 而是自建或蒸馏
- 最终目标: **本地推理为主 (95%), API 为辅 (5%)**

### Q: 新闻情绪因子的 IC 很低, 还有必要做吗?
有必要, 但要明确定位。A 股新闻情绪因子 IC 约 0.02-0.05, 排在资金和量价之后。关键价值:
1. **恐慌期效果显著**: 散户主导+卖空受限, 市场低迷时月超额 ~1.62% (乐观期仅 0.31%)
2. **事件驱动**: 突发事件 (降准/黑天鹅) 时, 新闻情绪对次日走势有即时反应能力
3. **边际成本趋零**: 蒸馏到本地后推理成本为 0, 即使 IC 偏低也是"免费午餐"

建议: 保留新闻情绪维度, 但设置自动降权机制 (连续 30 天 IC < 0.01 → 权重降至 0)。详见本文档"情绪因子效果评估"章节。
