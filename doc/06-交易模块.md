# 交易模块 (src/trading/)

## 概述

对接迅投 QMT 的模拟盘和实盘交易。

交易模块是整个系统的最后一环: 接收 `StrategyOrchestrator` 输出的操作清单 (`List[ActionItem]`)，通过 QMT API 执行买卖。

## 模块结构

| 文件 | 职责 |
|------|------|
| `qmt_trader.py` | QMT 交易客户端封装 |
| `order_manager.py` | 委托管理 (下单/撤单/记录) |
| `position_manager.py` | 持仓管理 (查询/快照) |
| `risk_control.py` | 风控模块 (仓位/止损/止盈) |
| `paper_trading.py` | 模拟盘引擎 |
| `live_trading.py` | 实盘引擎 (继承模拟盘 + 安全确认) |
| `trade_log.py` | 每日绩效记录 |

## 完整交易流程 (v5.0)

```
每日 09:15 — 系统自动执行:

1. 查询 QMT 持仓
   trader.query_positions() → List[HoldingPosition]
        │
2. 调用策略编排器
   orchestrator.execute(
       trade_date, holdings, total_capital, available_cash, price_map
   ) → List[ActionItem]
        │
3. 先执行卖出操作 (09:30 开盘)
   for action in sell_actions:
       trader.order_stock(action.code, "sell", action.target_quantity)
        │
4. 等待卖出成交, 确认资金释放
   (约 10:00)
        │
5. 再执行买入操作
   for action in buy_actions:
       trader.order_stock(action.code, "buy", action.target_quantity)
        │
6. 收盘后 (15:30)
   - 记录持仓快照
   - 计算日收益
   - 推送通知
```

### 为什么先卖后买?

- A 股 T+1 规则: 今日买入无法今日卖出
- 先卖释放资金 → 才有钱买新标的
- 卖出清单由 PositionMonitor (止损/止盈) + 策略卖出信号共同决定
- 买入清单由 PositionSizer 分配仓位

## QMT 交易 API

参照 [xttrader 官方文档](https://dict.thinktrader.net/nativeApi/xttrader.html)。

```python
trader = QMTTrader(
    account_id="your_account",
    mini_qmt_path="C:/MiniQMT/userdata_mini"
)
trader.connect()

# 下单 (order_type → xtconstant.STOCK_BUY / STOCK_SELL)
order_id = trader.order_stock(
    "600519.SH", "buy", 100,
    price_type="LATEST_PRICE"  # 最新价下单 (推荐)
)

# 查询资产
asset = trader.query_asset()
# → {total_asset, cash, market_value, frozen_cash}

# 查询持仓
positions = trader.query_positions()
# → [{code, volume, can_use_volume, avg_price, market_value, ...}]

# 查询委托
orders = trader.query_orders()
# → [{order_id, code, direction, status, filled_qty, ...}]

# 撤单
trader.cancel_order(order_id)
```

### 下单方式

| price_type | 说明 | 适用场景 |
|-----------|------|---------|
| `LATEST_PRICE` | 最新价 | 默认, 最常用 |
| `FIX_PRICE` | 指定价格 | 限价单 |
| `MARKET_SH_CONVERT_5_CANCEL` | 沪市五档即成转撤 | 大单快速成交 |
| `MARKET_SZ_CONVERT_5_CANCEL` | 深市五档即成转撤 | 同上 |
| `MARKET_PEER_PRICE_FIRST` | 对手方最优 | 市价单 (推荐) |

## 风控体系

### 多层风控设计

```
Layer 1: 策略级 (Signal 字段)
    每个 Signal 自带 stop_loss_pct / take_profit_pct / max_hold_days
         │
Layer 2: PositionMonitor (持仓级)
    扫描所有持仓 → 止损/止盈/移动止损/超期清仓
         │
Layer 3: SignalArbiter (组合级)
    max_holdings / max_buy_per_day / max_sell_per_day
         │
Layer 4: PositionSizer (资金级)
    max_single_pct / max_total_pct / min_trade_amount
         │
Layer 5: 交易模块风控 (执行级)
    日亏损限额 / 二次确认 (实盘)
```

### 风控参数

| 风控规则 | 环境变量 | 默认值 | 说明 |
|---------|---------|--------|------|
| 硬止损 | `PM_STOP_LOSS_PCT` | -8% | 亏损触发卖出 |
| 止盈 | `PM_TAKE_PROFIT_PCT` | 15% | 盈利触发卖出 |
| 移动止损 | `PM_TRAILING_STOP_PCT` | 5% | 从最高点回撤触发 |
| 超期清仓 | `PM_MAX_HOLD_DAYS` | 10 天 | 到期且未盈利则卖出 |
| 分批止盈 | `PM_PARTIAL_TAKE_PROFIT` | true | 止盈时只卖一半 |
| 最大持仓数 | `ARB_MAX_HOLDINGS` | 5 只 | 分散风险 |
| 单票上限 | `SIZER_MAX_SINGLE_PCT` | 20% | 单只不超总资产 20% |
| 总仓位上限 | `SIZER_MAX_TOTAL_PCT` | 80% | 留 20% 现金 |
| 日亏损限额 | (交易模块) | -5% | 日内亏损达标停止交易 |

### 风控优先级

当多个风控规则同时触发时:

```
硬止损 (score=100)  >  移动止损 (90)  >  止盈 (80)  >  超期清仓 (60)
```

score 越高 → 在 SignalArbiter 中排序越靠前 → 优先执行。

## 模拟盘 vs 实盘

### 模拟盘

```python
paper = PaperTrading(trader)
paper.execute_signals(action_items)  # 直接执行
```

- 使用迅投模拟盘账户
- 无资金风险
- 建议先用模拟盘运行 1~3 个月验证

### 实盘

```python
live = LiveTrading(trader)
live.confirm_live_mode()             # 必须二次确认!
live.execute_signals(action_items)   # 确认后执行
```

- `account_type = "live"` 区分日志和快照
- 需要显式调用 `confirm_live_mode()` 才能执行
- 建议: 模拟盘跑出稳定收益后再切实盘

### 从模拟盘到实盘的检查清单

| 序号 | 检查项 | 标准 |
|------|--------|------|
| 1 | 模拟盘运行时间 | ≥ 1 个月 |
| 2 | 回测夏普比率 | ≥ 1.0 |
| 3 | 模拟盘胜率 | ≥ 45% |
| 4 | 最大回撤 | ≤ -20% |
| 5 | 盈亏比 | ≥ 1.5 |
| 6 | 风控参数已配置 | 止损/止盈/仓位 |
| 7 | .env TRADING_PAPER_MODE | 改为 false |

## 每日操作节奏

| 时间 | 操作 | 说明 |
|------|------|------|
| 09:00 | MiniQMT 客户端开启 | 确认连接正常 |
| 09:15 | 调用 `POST /strategy/execute` | 获取操作清单 |
| 09:30 | 执行卖出 | 止损/止盈单优先 |
| 10:00 | 确认卖出成交 | 等待资金释放 |
| 10:00~10:30 | 执行买入 | 按清单依次下单 |
| 14:50 | 检查持仓 | 确认无遗漏 |
| 15:00 | 收盘 | 系统自动记录持仓快照 |
| 15:30 | 自动日报 | 推送当日损益和持仓变动 |
| 17:00 | 自动增量同步 | 更新行情和因子数据 |

## 常见问题

### Q: 下单失败怎么办?
检查: (1) MiniQMT 是否运行; (2) 资金是否充足; (3) 股票是否停牌; (4) 是否超过涨跌停价。所有下单失败会记录到 `trade_order` 表。

### Q: 如何处理部分成交?
`query_orders()` 会返回 `filled_qty`。如果部分成交，系统会在下次执行时重新评估。

### Q: 网络断开怎么办?
QMT 下单是本地客户端代理，即使 API 服务断开，已提交的委托仍会由 MiniQMT 执行。重连后查询最新状态即可。
