# 策略管理模块 (src/strategy/)

## 概述

三档策略引擎 + 完整买卖体系 + 持仓监控 + 信号仲裁 + 仓位分配，实现从信号生成到操作清单的全流程自动化。

专为 **A 股散户 T+1 低频交易** 设计。

## 整体架构

```
  策略信号生成 (三档 × 10 个策略)
         │ 每个策略同时输出 buy + sell 信号
         ▼
  持仓监控 (PositionMonitor)
         │ 扫描已有持仓 → 止损/止盈/移动止损/超期卖出
         ▼
  信号仲裁 (SignalArbiter)
         │ 去重 / 冲突解决 / T+1 校验 / 投票加分 / 持仓感知
         ▼
  仓位分配 (PositionSizer)
         │ 等权/ATR反比/凯利 分配, 100 股整手
         ▼
  最终操作清单 (List[ActionItem])
         │ 先卖后买, 按优先级排序
         ▼
  回测引擎 / QMT 交易执行
```

## 模块文件

| 文件 | 职责 |
|------|------|
| `base.py` | Signal / HoldingPosition / ActionItem 数据类 + BaseStrategy 基类 |
| `registry.py` | 策略注册表 (单例) + @register_strategy 装饰器 |

| `position_monitor.py` | 持仓监控 (止损/止盈/移动止损/超期清仓) |
| `signal_arbiter.py` | 多策略信号仲裁 (去重/T+1/仓位约束) |

| `orchestrator.py` | 策略编排器 v2 (信号→监控→仲裁→仓位→操作清单) |

## 统一信号协议 (Signal)

所有策略输出标准 Signal，包含风控字段：

```python
@dataclass
class Signal:
    trade_date: date          # 交易日
    code: str                 # QMT 格式 000001.SZ
    direction: str            # "buy" / "sell" / "hold"
    score: float              # 排序打分, 越高越优先
    quantity: int             # 建议数量
    strategy_name: str        # 策略标识
    strategy_tier: str        # "rule" / "scoring" / "ml" / "risk"
    reason: str               # 可读理由

    # 风控字段 (策略建议, PositionMonitor 使用)
    stop_loss_pct: float      # 止损 (如 -8.0 = 亏 8% 触发)
    take_profit_pct: float    # 止盈 (如 15.0 = 赚 15% 触发)
    max_hold_days: int        # 最大持仓天数
    trailing_stop_pct: float  # 移动止损 (从最高回撤 N%)
    min_amount: float         # 最低日成交额 (万元)
```

默认值全部来自 `.env` 中 `SIG_*` 参数。

## Tier 1: 规则策略 (7 个)

### 1. 动量策略 (momentum)

**核心思想**: 趋势追随——强者恒强，买入近期涨幅最大的股票。

**适用场景**:
- 牛市或趋势上涨阶段
- 市场情绪高涨、成交量活跃
- 热点板块轮动清晰

**不适用场景**:
- 震荡市（容易追高被套）
- 熊市（趋势反转频繁）

**信号逻辑**:
- **买入**: 计算 universe 中所有股票近 `lookback_days` 日涨幅，按涨幅排序取 Top N
- **卖出**: 持仓股票的动量反转（lookback 收益率 < 0）

**关键参数** (`.env`):

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_MOM_LOOKBACK` | 20 | 回看天数 (20=月度动量) |
| `STRAT_MOM_TOP_N` | 10 | 选取涨幅前 N 只 |
| `STRAT_MOM_MIN_TURNOVER` | 0.5 | 最低日均换手率 (%, 过滤冷门股) |
| `STRAT_MOM_STOP_LOSS` | -8.0 | 止损 (%) |
| `STRAT_MOM_TAKE_PROFIT` | 15.0 | 止盈 (%) |
| `STRAT_MOM_MAX_HOLD_DAYS` | 5 | 最大持仓天数 (短线) |
| `STRAT_MOM_TRAILING_STOP` | 5.0 | 移动止损 (%) |

**调参建议**:
- 牛市初期用较长 lookback (30~60)，捕捉中期趋势
- 快速轮动市场用较短 lookback (5~10)
- `min_turnover` 过滤不活跃标的，避免流动性风险

---

### 2. 反转策略 (reversal)

**核心思想**: 均值回归——跌多了会反弹，买入近期超跌股票。

**适用场景**:
- 底部复苏阶段
- 恐慌抛售后的技术反弹
- 大盘急跌企稳后

**不适用场景**:
- 持续下跌的熊市（"接飞刀"风险）
- 个股基本面恶化的下跌

**信号逻辑**:
- **买入**: 计算近 `lookback_days` 跌幅，剔除跌幅超过 `max_drawdown` 的（避免接飞刀），取跌幅最大的 Top N
- **卖出**: 持仓盈利达到 `bounce_target_pct` (反弹目标)

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_REV_LOOKBACK` | 10 | 回看天数 |
| `STRAT_REV_TOP_N` | 10 | 选取跌幅前 N 只 |
| `STRAT_REV_MAX_DRAWDOWN` | -30.0 | 最大跌幅过滤 (跌超 30% 不选) |
| `STRAT_REV_BOUNCE_TARGET` | 5.0 | 反弹卖出目标 (%, 赚 5% 即卖) |
| `STRAT_REV_STOP_LOSS` | -5.0 | 止损 (%, 更紧, 防止继续跌) |
| `STRAT_REV_MAX_HOLD_DAYS` | 5 | 最大持仓天数 (短线) |

**调参建议**:
- `max_drawdown` 是安全阀：设太松（-50%）容易接飞刀，太紧（-10%）机会太少
- 反弹策略天然持仓短，`bounce_target_pct` 建议 3%~8%
- 与动量策略对冲使用效果更好

---

### 3. 行业轮动策略 (industry_rotation)

**核心思想**: 资金在行业间流动——买入强势行业的龙头股。

**适用场景**:
- 结构性行情（板块分化明显）
- 政策驱动型行情
- 牛市中后期的板块轮动

**信号逻辑**:
- 计算每个行业的平均涨幅 → 选 Top N 强势行业
- 每个行业内选涨幅最大的龙头股
- **卖出**: 持仓股所在行业跌出 Top N

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_IND_LOOKBACK` | 20 | 行业涨幅计算周期 |
| `STRAT_IND_TOP_INDUSTRIES` | 3 | 选取强势行业数 |
| `STRAT_IND_STOCKS_PER_IND` | 3 | 每行业选几只龙头 |
| `STRAT_IND_MAX_HOLD_DAYS` | 10 | 最大持仓天数 |

**调参建议**:
- 行情快速轮动时减少 `LOOKBACK` (10)
- `STOCKS_PER_IND=1` 集中持仓龙头更激进

### 4. 均线突破策略 (moving_average)

**核心思想**: 经典双均线系统——短均线上穿长均线(金叉)买入，下穿(死叉)卖出。

**适用场景**:
- 温和上涨趋势
- 趋势明确的中长期行情
- 波动不大的蓝筹股

**不适用场景**:
- 震荡市（频繁假突破）
- 小盘股（波动太大，假信号多）

**信号逻辑**:
- **买入**: MA(short) 从下向上穿越 MA(long)，按突破强度排序
- **卖出**: MA(short) 从上向下穿越 MA(long)，仅对持仓股生效

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_MA_SHORT` | 5 | 短期均线天数 |
| `STRAT_MA_LONG` | 20 | 长期均线天数 |
| `STRAT_MA_TOP_N` | 15 | 金叉信号选前 N 只 |
| `STRAT_MA_STOP_LOSS` | -6.0 | 止损 (%) |

**调参建议**:
- (5, 20) 适合短线；(10, 60) 适合中线
- 可以和动量策略组合使用：均线确认趋势 + 动量选个股

### 5. T+1 宽网格策略 (grid_trading)

**核心思想**: 在股票的震荡区间内低买高卖。专门适配 A 股 T+1 规则。

**T+1 适配关键**: 买入和卖出发生在不同交易日。今天扫描买入机会（价格接近区间下沿），同时对已持仓接近上沿的标的生成隔日卖出信号。

**适用场景**:
- 箱体震荡行情
- 标的有明确的支撑位和压力位
- 波动率适中（太低赚不到差价，太高容易破位）

**信号逻辑**:
- 计算 `lookback_days` 内的最高价和最低价作为区间
- **买入**: 当前价格在区间底部 `grid_pct`% 范围内
- **卖出**: 持仓当前价格在区间顶部 `grid_pct`% 范围内

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_GRID_PCT` | 10.0 | 网格宽度 (%, 底部/顶部各 10% 触发) |
| `STRAT_GRID_LOOKBACK` | 60 | 区间计算回看天数 |
| `STRAT_GRID_STOP_LOSS` | -5.0 | 止损 (%, 破位立即止损) |
| `STRAT_GRID_MAX_HOLD_DAYS` | 5 | 最大持仓天数 |

**调参建议**:
- `GRID_PCT=5` 更激进（触发频率高但利润薄），`=15` 更保守
- 配合大盘指数 ETF 效果较好（波动有规律）
- 不适合趋势股（会在上沿过早卖出）

### 6. 双低可转债策略 (cb_dual_low)

**核心思想**: 可转债的"双低值"= 转债价格 + 转股溢价率，越低安全边际越高。

**适用场景**:
- 任何市场环境（可转债有债底保护）
- 特别适合熊市和震荡市
- 风险厌恶型投资者

**优势**: 可转债有纯债价值兜底，下跌空间有限（面值 100 附近），上涨跟随正股。

**信号逻辑**:
- 计算每只转债的双低值 = price × 价格权重 + premium × 溢价率权重
- 过滤：价格 > `max_price`、溢价率 > `max_premium`、评级 < `min_rating` 的剔除
- 选双低值最低的 Top N
- **卖出**: 持仓转债双低值升高超过阈值

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_CB_PRICE_WEIGHT` | 1.0 | 价格在双低公式中的权重 |
| `STRAT_CB_PREMIUM_WEIGHT` | 1.0 | 溢价率在双低公式中的权重 |
| `STRAT_CB_MAX_PRICE` | 130.0 | 最高价格 (元, >130 不买) |
| `STRAT_CB_MAX_PREMIUM` | 50.0 | 最高溢价率 (%) |
| `STRAT_CB_MIN_RATING` | AA- | 最低评级 |
| `STRAT_CB_STOP_LOSS` | -3.0 | 止损 (%, 转债止损松) |
| `STRAT_CB_MAX_HOLD_DAYS` | 20 | 持仓天数 (转债可持久) |
| `STRAT_CB_MIN_AMOUNT` | 1000.0 | 流动性 (万元, 转债门槛低) |

**调参建议**:
- 保守型：`MAX_PRICE=120, MAX_PREMIUM=30`
- 激进型：`MAX_PRICE=140, MAX_PREMIUM=80`（允许小幅溢价但低价格补偿）
- 可以定期（每周/两周）轮动：卖出双低值升高的，买入新低的

### 7. 低波红利策略 (low_vol_dividend)

**核心思想**: 选低波动 + 高股息的股票，牺牲进攻性换取稳定收益。

**适用场景**:
- 熊市防守
- 市场不确定性高
- 追求稳健现金流

**信号逻辑**:
- 计算年化波动率，过滤高波动标的
- 计算股息率（优先用实际数据，回退用 PE 估算）
- 综合评分 = (1 - 波动率归一) × 波动权重 + 股息率归一 × 股息权重
- **卖出**: 持仓股波动率飙升超过阈值

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_LV_LOOKBACK` | 60 | 波动率计算天数 |
| `STRAT_LV_MAX_VOL_PCT` | 40.0 | 最高年化波动率 (%) |
| `STRAT_LV_MIN_DIV_YIELD` | 2.0 | 最低股息率 (%) |
| `STRAT_LV_STOP_LOSS` | -5.0 | 止损 (%) |
| `STRAT_LV_MAX_HOLD_DAYS` | 20 | 持仓天数 (防守策略更久) |

**调参建议**:
- 纯防守：`MAX_VOL_PCT=25, MIN_DIV_YIELD=3`（极低波动+高分红）
- 平衡型：`MAX_VOL_PCT=40, MIN_DIV_YIELD=2`
- 适合与动量策略做资金对冲：牛市动量仓位大，熊市低波仓位大

## ETF 全球资产轮动策略 (etf_rotation)

**核心思想**: 在全球主要 ETF 资产类别 (A 股/美股/黄金/债券等) 之间, 基于动量信号进行月度轮动, 配合崩盘保护机制自动切换防御资产。

**模块结构** (`src/strategy/etf_rotation/`):

| 文件 | 职责 |
|------|------|
| `universe.py` | ETF 池管理 (风险池/防御池/哨兵池) |
| `momentum.py` | 动量评分 (13612W / R²×Return / DualMomentum) |
| `crash_guard.py` | 崩盘保护 (广度动量/绝对动量/波动率门控) |
| `rotator.py` | 轮动引擎 (排名/选择/反转过滤/信号生成) |
| `etf_rotation_strategy.py` | `ETFRotationStrategy` — 继承 BaseStrategy, 接入 Orchestrator |

**ETF 候选池** (16 只, 覆盖 5+ 资产类别):

| 分类 | 代表 ETF | 说明 |
|------|---------|------|
| A 股宽基 | 510300/159915/510500 | 沪深300/创业板/中证500 |
| 港股/美股 | 513180/513100/513500 | 恒生科技/纳指100/标普500 |
| 商品 | 518880/159985 | 黄金/豆粕 |
| 债券 (防御) | 511260/511010 | 十年国债/国债 |

**动量评分方法**:
- **13612W** (Keller 2017): 加权多周期动量, 近期权重更高
- **R²×Return**: 年化收益率 × 趋势拟合优度, 过滤毛刺行情
- **Dual Momentum** (Antonacci 2014): 相对+绝对动量双重确认

**崩盘保护**: 哨兵资产动量为负时自动增加防御仓位 (国债 ETF), 全部为负时 100% 切换至国债。

**关键参数** (`.env`):

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ETF_ROTATION_MOMENTUM_METHOD` | 13612w | 动量方法 |
| `ETF_ROTATION_TOP_K` | 2 | 每期持有 ETF 数 |
| `ETF_ROTATION_REBALANCE_INTERVAL` | 20 | 调仓间隔 (交易日) |
| `ETF_ROTATION_MIN_HOLD_DAYS` | 9 | 最小持有天数 (防反转) |
| `ETF_ROTATION_VOLATILITY_GATE` | true | 启用波动率门控 |

## 组合优化 (src/portfolio/)

### CAAOptimizer — Classical Asset Allocation

基于 Keller, Butler & Kipnis (2015) 论文, 动量驱动的纯多头均值-方差优化:

- **动量收益估计**: 1/3/6/12 月动量均值
- **纯多头约束**: 消除 MVO 不稳定性
- **权重上限**: 风险资产 ≤ 25%, 现金不设上限
- **CLA 求解**: Critical Line Algorithm

### RiskParityOptimizer — 等风险贡献

每个资产对组合风险的贡献相等, 不依赖收益预测, 稳健性好。

### RiskAttributor — 简化 Barra 风险归因

将组合收益分解为 alpha + 风格因子 + 行业暴露 + 残差:
- 5 大风格因子: size, value, momentum, volatility, liquidity
- 截面 OLS 回归, 支持行业哑变量

```python
from src.portfolio.optimizer import PortfolioOptimizer

optimizer = PortfolioOptimizer(method="caa")
weights = optimizer.optimize(prices, target_vol=0.10, cap=0.25)
```

## Tier 2: 多因子打分策略

**核心思想**: 多维度评价股票质量，综合打分排序选股。

### 等权打分 (multifactor_equal)

所有因子标准化后等权求和。适合因子研究初期，不确定哪个因子更好时使用。

### IC 加权打分 (multifactor_ic)

按各因子的滚动 IC (Information Coefficient) 均值加权。IC 高的因子权重大——让数据决定权重。

**适用场景**:
- 拥有多个已验证因子
- 中频调仓（周度 / 月度）
- 追求系统化投资

**信号逻辑**:
1. 从数据库加载截面因子数据
2. MAD 去极值 → Z-Score 标准化
3. 根据模式计算因子权重（等权 or IC 加权）
4. 加权求和 → 排序 → 买入 Top N
5. **卖出**: 持仓股因子排名下降至后 30%

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_SCORE_IC_WINDOW` | 20 | IC 滚动计算窗口 (交易日) |
| `STRAT_SCORE_TOP_N` | 20 | 选取综合得分前 N 只 |
| `STRAT_SCORE_NEUTRALIZE` | false | 是否行业中性化 |
| `STRAT_SCORE_STOP_LOSS` | -8.0 | 止损 (%) |
| `STRAT_SCORE_MAX_HOLD_DAYS` | 10 | 最大持仓天数 |

**因子配置方式**: 策略实例化时通过 `config.factor_names` 指定使用的因子列表，或在数据库 `strategy` 表的 `factor_names_json` 字段配置。

**训练/使用流程**:
1. 在因子工程模块完成单因子 IC 检验
2. 选择 IC > 0.02 的因子放入 `factor_names`
3. 选择权重模式：新手用 equal，有足够历史数据后用 ic
4. 通过 API 创建策略并绑定标的池
5. 每日自动生成信号

## Tier 3: ML 策略

**核心思想**: 用机器学习模型预测个股未来收益率，选预测值最高的买入。

### LightGBM 策略 (lgb_ml)

**适用场景**:
- 因子数量多（20+），人工打分效率低
- 非线性因子组合效果更好
- 有足够历史数据训练（至少 1 年日频）

**信号逻辑**:
1. 加载截面因子数据
2. 模型预测每只股票的未来收益率
3. 按预测值排序 → 高于 `long_threshold` 的买入 Top N
4. **卖出**: 持仓股预测收益低于阈值

### XGBoost 策略 (xgboost_ml) / CatBoost 策略 (catboost_ml)

预留接口，实现 `_create_model()` 即可切换。当前暂用 LGB 替代。

**关键参数**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `STRAT_ML_TOP_N` | 10 | 选取预测收益前 N 只 |
| `STRAT_ML_LONG_THRESHOLD` | 0.0 | 做多阈值 (预测值 > 此值才买) |
| `STRAT_ML_STOP_LOSS` | -8.0 | 止损 (%) |
| `STRAT_ML_MAX_HOLD_DAYS` | 8 | 最大持仓天数 |

**ML 标签周期**: `.env` 中 `ML_LABEL_PERIOD=2`（隔日收益），适合 T+1 散户。

**训练流程**:

```
1. 因子数据准备
   400+ QMT 因子 + 自定义技术因子
         │
2. 单因子 IC 筛选
   IC > 0.02 的入选 (约 30 个)
         │
3. 数据集构建
   X = 截面因子矩阵
   y = 未来 N 日收益率 (ML_LABEL_PERIOD)
         │
4. 时序分割
   60% 训练 / 20% 验证 / 20% 测试
         │
5. LightGBM 训练
   early_stopping + 验证集
         │
6. 模型评估
   IC / ICIR / 分组收益 / 夏普比率
         │
7. 自动迭代 (AutoIterateEngine)
   因子进化 → 重训 → 评分 → 循环
         │
8. 最优模型入库
   保存至 ./models/, 注册到策略池
```

**自动迭代详解**: 见 [03-机器学习模块](03-机器学习模块.md)

## 持仓监控 (PositionMonitor)

实时扫描持仓，按优先级触发 4 种卖出信号：

| 优先级 | 类型 | 条件 | score |
|--------|------|------|-------|
| 1 | 硬止损 | 亏损 ≥ `PM_STOP_LOSS_PCT` | 100 |
| 2 | 移动止损 | 盈利中从最高回撤 ≥ `PM_TRAILING_STOP_PCT` | 90 |
| 3 | 止盈 | 盈利 ≥ `PM_TAKE_PROFIT_PCT` | 80 |
| 4 | 超期清仓 | 持仓天数 ≥ `PM_MAX_HOLD_DAYS` 且未盈利 | 60 |

**T+1 约束**: 今天买入的仓位不会生成卖出信号。

**分批止盈**: 设置 `PM_PARTIAL_TAKE_PROFIT=true`，止盈时只卖出 `PM_PARTIAL_TP_RATIO`（默认 50%）。

## 信号仲裁 (SignalArbiter)

解决多策略信号冲突问题：

| 规则 | 说明 |
|------|------|
| **Regime 门控** | 根据市场 drift/volatility 状态, 仅放行适合当前 regime 的策略信号 (P1-35) |
| 去重合并 | 同一标的被多策略推荐 → 投票加分 |
| 冲突解决 | 同一标的既有 buy 又有 sell → 卖出优先 |
| T+N 校验 | 按资产类型交易规则校验 (A 股 T+1, 跨境 ETF T+0, 可转债 T+0 等) |
| 持仓感知 | 已持有的不重复买入，不持有的不卖出 |
| 仓位约束 | 总持仓不超过 `ARB_MAX_HOLDINGS`，卖出释放的槽位才允许买入 |
| 买卖限额 | 每日买入 ≤ `ARB_MAX_BUY_PER_DAY`，卖出 ≤ `ARB_MAX_SELL_PER_DAY` |
| **换手率约束** | 当日买卖总金额 / 总资产 ≤ `ARB_MAX_DAILY_TURNOVER_PCT` (默认 20%)，超限时截断低优先级买入信号 |

**换手率约束**: 高换手率侵蚀 alpha (A 股双边费率 ~0.15%，年化换手率 200% 意味着 30% 费率成本)。`_apply_turnover_constraint()` 按信号优先级排序，累计买入金额超过日限额时截断剩余买入，卖出信号不受影响。

**投票加分**: 同一标的被 N 个策略看好，综合分 = base × (1 + `ARB_MULTI_BONUS` × (N-1))。

### Regime 门控 (P1-35, 研究驱动)

基于 Drift Regime Gating (arXiv:2511.12490, 2025) 和 Agentic Factor Investing (arXiv:2603.14288, 2026), 在信号级别根据实时市场 regime 决定是否放行:

| Regime | 检测方式 | 放行策略 | 拦截策略 |
|--------|---------|---------|---------|
| **drift_up** (上涨趋势) | 63日内上涨日占比 >60% | 动量/行业轮动/ML/ETF轮动 | 反转 |
| **drift_down** (下跌趋势) | 63日内上涨日占比 <40% | 反转/双低/低波/ETF轮动 | 动量/行业轮动 |
| **high_vol** (高波动) | 20日年化波动率 >80 分位 | 反转/双低/低波 | 动量/网格 |
| **normal** (常态) | 以上均不满足 | 所有策略 | 无 |

与现有 `macro_env.py` 的关系: macro_env 是"粗粒度仓位调节" (6 种状态 → 仓位倍数), Regime 门控是"细粒度信号过滤" (4 种状态 → 逐信号放行/拦截), 两者互补。

## 仓位分配 (PositionSizer)

四种分配模式：

### 等权模式 (equal)
每只股票平均分配。最简单，散户推荐。

### ATR 反比模式 (atr)
波动率大的仓位小，波动率小的仓位大。适合混合不同波动率的标的。

### 分数凯利模式 (kelly) — 研究驱动改进 (P1-32)
f* = kelly_frac × (bp - q) / b，其中 b = 盈亏比，p = 估计胜率，kelly_frac 为分数系数。

**关键改进**: 默认使用 **1/4 Kelly** (kelly_frac=0.25) 而非完整 Kelly:
- 完整 Kelly 波动过大, 参数估计误差会导致过度集中 → 爆仓风险
- **半 Kelly (0.5)**: ~75% 最优增长率, 波动降一半 (学术黄金标准, Ziemba 2003)
- **1/4 Kelly (0.25)**: 更保守, 适合散户参数不确定场景
- 学术依据: Kelly (1956), Thorp (2006), MacLean (2010)

### CAA 模式 (caa)
基于 Keller (2015) CAA 动量驱动均值-方差优化, 或 Riskfolio-Lib 风险预算。详见 P1-05, P1-36。

### 回撤自适应仓位缩减 (DrawdownGuard, P1-32)

当组合回撤超过阈值时, 自动缩减总仓位上限:

| 回撤幅度 | 仓位缩至 | 说明 |
|----------|---------|------|
| < -3% | 90% | 轻微回调, 小幅减仓 |
| < -5% | 75% | 中等回撤, 显著减仓 |
| < -8% | 50% | 较大回撤, 半仓防守 |
| < -12% | 25% | 严重回撤, 轻仓 |
| < -15% | 10% | 极端回撤, 几乎空仓 |

回撤自适应与宏观仓位倍数叠加: 最终仓位上限 = DrawdownGuard × macro_scale × SIZER_MAX_TOTAL_PCT

**约束**:
- 单票不超过 `SIZER_MAX_SINGLE_PCT` (默认 20%)
- 总仓位不超过 `SIZER_MAX_TOTAL_PCT` × 宏观调节系数 × 回撤缩减系数
- 最小 100 股 (`SIZER_LOT_SIZE`), 科创板 200 股 (按资产类型交易规则)
- 最小交易金额 `SIZER_MIN_TRADE_AMOUNT` (默认 5000 元)

## 散户实战建议

### 推荐配置

```
总资金: 50万~100万
持仓数: 3~5 只 (ARB_MAX_HOLDINGS=5)
单票上限: 20% (SIZER_MAX_SINGLE_PCT=20)
每日新买: 1~2 只 (ARB_MAX_BUY_PER_DAY=2)
止损: -8% (PM_STOP_LOSS_PCT=-8)
止盈: +15% (PM_TAKE_PROFIT_PCT=15)
盈亏比: 15/8 = 1.875 > 1.5 ✓
```

### 不同市场环境的策略选择

| 你判断的市场 | 操作建议 |
|-------------|---------|
| "现在是牛市" | 设 macro_state=bull_strong，动量+行业轮动为主 |
| "震荡看不清方向" | 设 range_bound，网格+双低+低波为主 |
| "感觉要跌了" | 设 bear_mild，双低+低波，仓位自动降到 50% |
| "跌完了想抄底" | 设 recovery，反转+双低，仓位 90% |

### 操作节奏

```
每日 09:15  → 调用 /execute 获取操作清单
     09:30  → 按优先级执行卖出
     10:00  → 确认卖出成交后执行买入
     14:50  → 检查持仓,确认无遗漏
```
