# qt-quant 用户手册

面向第一次接触本项目的用户：从安装到选股、模拟交易、回测、QMT 下单的**常用命令**与推荐工作流。

> 详细设计见 [00-总体设计](00-总体设计.md) 及各模块文档；本手册侧重「怎么跑起来」。

---

## 1. 项目是做什么的

qt-quant 是一套 A 股量化工具链，典型流程如下：

```
数据采集 / QMT 同步  →  PostgreSQL (日线 K 线)
        ↓
   MA 程序选股 (CLI 或 Web UI)
        ↓
   选股清单 JSON (reports/candidates_*.json)
        ↓
   回测验证  /  Web 模拟盘练习  /  QMT 模拟·实盘下单
```

| 能力 | 是否依赖 QMT | 入口 |
|------|-------------|------|
| 数据采集、日线入库 | 可选（多源 fallback） | `scripts/`、`src/datacollect/` |
| CLI 选股 | 否 | `python -m src.selection` |
| **Web 模拟盘 UI** | **否** | `python -m src.webui` |
| 策略回测 | 否 | `scripts/run_backtest.py` |
| QMT 模拟/实盘交易 | **是** | `python -m src.trading` |

## 2. 环境与安装

### 2.1 前置条件

- **Python 3.10+**
- **PostgreSQL**（存 K 线、模拟盘账户等）
- （可选）**迅投 Mini QMT** 终端 — 仅 QMT 同步 / 实盘·QMT 模拟盘时需要
- 推荐包管理：**uv** 或已有虚拟环境 `.venv`

### 2.2 安装依赖

在项目根目录（含 `pyproject.toml` 的目录）执行：

```bash
# 推荐
uv sync

# 或使用已有 venv
.venv\Scripts\python.exe -m pip install -e .
```

Windows PowerShell 下后续命令可统一写为：

```powershell
$py = ".venv\Scripts\python.exe"
```

### 2.3 配置环境变量

复制模板并按需填写：

```bash
# 数据库（必填）
copy env\.env.example env\.env.db
# 编辑 DATABASE_URL=postgresql://user:pass@localhost:5432/qt_quant

# QMT（仅 QMT 交易/同步时需要）
copy env\.env.example env\.env.qmt
# 编辑 QMT_PATH、QMT_ACCOUNT_ID 等
```

主要变量见 `env/.env.example`：`DATABASE_URL`、`QMT_PATH`、`SELECTION_STRATEGY` 等。

### 2.4 初始化数据库

```bash
uv run python scripts/init_db.py
```

Web 模拟盘首次启动时还会自动创建 `paper_users` / `paper_accounts` / `paper_trades` 表（与业务 K 线表隔离）。

## 3. 数据准备（选股/回测的前置）

选股与回测都依赖 PostgreSQL 中的 **日线数据**（`stock_daily`、`etf_daily` 等）。数据不全时选股结果会偏少或为空。

### 3.1 定时增量同步（推荐）

本地常驻进程，按配置在收盘后同步 K 线：

```bash
uv run python scripts/run_scheduler.py
```

默认同步时间见 `config/app.json` / `SCHEDULER_DAILY_SYNC_TIME`（如 16:30）。

### 3.2 手动同步脚本（示例）

| 脚本 | 用途 |
|------|------|
| `scripts/sync_qmt_all_available.py` | 从 QMT 批量同步可用日线 |
| `scripts/sync_hk_stocks_from_qmt.py` | 港股通相关 |
| `scripts/etf_daily_coverage_and_align.py` | ETF 日线对齐 |
| `uv run python -m src.data.inst_holder_sync --report-date 2026-03-31` | 机构持有家数（东财）落库 `stock_inst_holder` |
| `uv run python -m src.data.inst_holder_sync --year 2026` | 同步指定年份全部已到达报告期 |

Web UI 选股时的机构家数 **优先读库**；库中无当年完整数据时再 live 抓取东财/新浪。

具体参数见各脚本文件头注释；数据模块说明见 [01-数据模块](01-数据模块.md)、[02-数据采集模块](02-数据采集模块.md)。

### 3.3 检查数据是否就绪

```bash
uv run python scripts/run_screen_audit.py --date 2026-07-09
```

不加 `--audit` 时会跑一遍初筛并输出候选数量；加 `--audit` 可看各过滤阶段通过率。

## 4. 选股

当前 Web UI 与 CLI 默认策略为 **启动突破**（`bull_launch`）。策略参数预设见 `config/strategies/bull_launch.json`；Web UI 中可覆盖部分参数（存浏览器本地）。

### 4.1 命令行选股（适合脚本/盘后批处理）

```bash
# 指定交易日初筛，输出 JSON
uv run python -m src.selection screen --date 2026-07-09

# 同时输出 CSV
uv run python -m src.selection screen --date 2026-07-09 --csv

# 指定策略 preset（需 JSON 存在；Python 实现须已注册）
uv run python -m src.selection screen --date 2026-07-09 --strategy bull_launch --csv

# 指定输出路径
uv run python -m src.selection screen --date 2026-07-09 --output reports/my_picks.json
```

**输出位置**（默认）：`reports/candidates_{策略ID}_{YYYYMMDD}.json`

清单内包含：`candidates`（代码列表）、`ma_snapshots`（指标快照）、`export_shortlist` 等。

### 4.2 选股审计（看漏斗、排查为何候选少）

```bash
uv run python scripts/run_screen_audit.py --date 2026-07-09
uv run python scripts/run_screen_audit.py --date 2026-07-09 --audit
uv run python scripts/run_screen_audit.py --date 2026-07-09 --rank-dist
```

### 4.3 Web UI 选股（适合交互、模拟挂单）

见 [第 5 节](#5-web-模拟盘-ui)；在「选股」视图点 **每日选股** / **每日选基**，可调参数后批量挂单。

策略切换与参数说明见 [06-模拟盘交易UI](06-模拟盘交易UI.md)。

## 5. Web 模拟盘 UI

**不依赖 QMT**：本地浏览器里的模拟交易终端，支持按历史交易日回放、多用户、华泰费率。

### 5.1 启动

```bash
uv run python -m src.webui
# 或
.venv\Scripts\python.exe -m src.webui --port 8001
```

可选参数：

| 参数 | 说明 |
|------|------|
| `--port 8001` | 监听端口（默认 8001） |
| `--host 0.0.0.0` | 允许局域网访问（默认） |
| `--reload` | 开发热重载 |

启动后终端会打印：

- 电脑：`http://127.0.0.1:8001`
- 手机（同一 WiFi）：`http://<本机局域网IP>:8001`

### 5.2 登录

| 项 | 值 |
|----|-----|
| 默认用户 | `root` |
| 默认密码 | `1234` |

也可在页面 **注册** 新用户（仅用户名 + 口令）。每个用户模拟资金、持仓、成交记录相互隔离。

### 5.3 界面功能速览

| 区域 | 功能 |
|------|------|
| 顶部资产条 | 总资产、当日/总盈亏、可用资金、改资金、重置、退出 |
| 交易日条 | 切换历史交易日（日历）、用于按日回放 |
| **交易** | 手动买入/卖出；价格在当日最高–最低区间内选择 |
| **选股** | 选策略 → 每日选股/选基 → 列表立即显示（机构家数转圈）→ 优先读 DB ``stock_inst_holder`` → 后台多源补全 → 勾选挂单 |
| 持仓 / 委托 / 成交 | 与券商终端类似的列表；成交可逐条删除记录 |

### 5.4 推荐练习流程（手动回测感）

1. 启动 UI 并登录。
2. 将 **交易日** 切到某一历史日期（需库中有该日 K 线）。
3. **选股** → 运行「每日选股」→ 勾选标的 → 填挂单价/数量 → **批量挂单**。
4. 点 **下一交易日**，观察委托是否按规则成交（选股挂单：当日最低价 < 挂单价则成交）。
5. 在 **持仓** 查看浮动盈亏；在 **交易** 面板卖出。

撮合与费率规则详见 [06-模拟盘交易UI](06-模拟盘交易UI.md)。

### 5.5 健康检查

```bash
curl http://127.0.0.1:8001/health
# {"status":"ok"}
```

## 6. 回测

回测用于在投入实盘前，用历史数据检验「按选股清单买卖」的效果。

### 6.1 基于选股清单 JSON（最常用）

先完成 [第 4 节](#4-选股) 得到 `reports/candidates_bull_launch_YYYYMMDD.json`，再：

```bash
uv run python scripts/run_backtest.py ^
  --picks-file reports/candidates_bull_launch_20260709.json ^
  --start 2025-01-01 ^
  --end 2025-12-31 ^
  --capital 1000000
```

| 参数 | 说明 |
|------|------|
| `--picks-file` | 单个 candidates JSON |
| `--picks-dir reports` | 目录下多个 `candidates_*.json` |
| `--screen` | 不回读文件，在区间内**每日现场 MA 初筛**（较慢） |
| `--start` / `--end` | 回测区间 |
| `--capital` | 初始资金 |

### 6.2 单笔交易费用回测（CLI）

适合验证某只股票买卖一笔的盈亏与费用明细：

```bash
uv run python -m src.backtest.cli trade 600519 ^
  --buy 2025-01-02 --sell 2025-06-30 --qty 100

uv run python -m src.backtest.cli info 600519
```

港股通示例：`00700` 或 `HK00700`。

更多指标与陷阱说明见 [04-回测引擎](04-回测引擎.md)。

## 7. QMT 交易（可选）

需要本机安装并登录 **Mini QMT**，且 `env/.env.qmt` 配置正确。

### 7.1 按选股清单批量下单

```bash
# QMT 模拟盘
uv run python -m src.trading execute ^
  --picks reports/candidates_bull_launch_20260709.json ^
  --mode paper

# 实盘（会要求确认，慎用）
uv run python -m src.trading execute ^
  --picks reports/candidates_bull_launch_20260709.json ^
  --mode live
```

### 7.2 与 Web 模拟盘的区别

| | Web 模拟盘 (`src/webui`) | QMT 模拟/实盘 (`src/trading`) |
|--|--------------------------|-------------------------------|
| 依赖 QMT | 否 | 是 |
| 数据 | PostgreSQL 日线 + 内存撮合 | QMT 柜台 |
| 用途 | 练习 UI、按日回放、选股挂单 | 对接真实/QMT 模拟柜台 |

市场规则与支持的板块见 [05-交易模块](05-交易模块.md)。

## 8. 策略与配置

### 8.1 策略 preset 文件

| 策略 ID | 文件 | 说明 |
|---------|------|------|
| `bull_launch` | `config/strategies/bull_launch.json` | 启动突破（当前 UI 已注册） |
| `bear_rebound` | `config/strategies/bear_rebound.json` | 熊市反弹（JSON 预留，待实现注册） |

### 8.2 切换策略的方式

```bash
# 环境变量
set SELECTION_STRATEGY=bull_launch

# CLI
uv run python -m src.selection screen --strategy bull_launch --date 2026-07-09

# 全局默认
# 编辑 config/app.json → selection.active_strategy
```

Web UI 选股页下拉框仅显示 **已在代码中注册** 的策略（当前为启动突破）。

### 8.3 全局应用配置

`config/app.json`：调度时间、回测滑点、风控默认值等。  
密钥与连接串：`env/.env.*`，不要提交到 Git。

## 10. 典型一日工作流（盘后）

```text
1. 确保 K 线已同步（scheduler 或手动 sync）
2. 选股
     CLI:  python -m src.selection screen --date <最新交易日> --csv
     或 UI:  打开 Web UI → 选股 → 每日选股
3. 验证
     python scripts/run_backtest.py --picks-file reports/candidates_....json
     或 UI 按日回放 + 模拟挂单
4. （可选）QMT 模拟盘 execute --mode paper
```

## 11. 故障排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 选股 0 只 | 当日无 K 线 / 条件过严 | `run_screen_audit.py --audit`；放宽 UI 或 JSON 参数 |
| Web UI 无法登录 | DB 未连上 | 检查 `DATABASE_URL`、PostgreSQL 是否运行 |
| 切换交易日无行情 | 该日该代码无日线 | 换日期或补数据 |
| QMT execute 失败 | 终端未开 / 路径错误 | 检查 `QMT_PATH`、账户登录 |
| 回测报错 | 清单日期与 `--start/--end` 不匹配 | 确认 JSON 内 `trade_date` 与区间 |

日志级别：`LOG_LEVEL=DEBUG`（见 `env/.env.trading` 或环境变量）。
