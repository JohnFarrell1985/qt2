# Fin-R1 数据访问能力指南

本文档详细说明 vLLM 和 api-middleware 如何通过提示词读取数据库中的全部数据。

## 📊 数据访问架构

```
用户提问 (Web UI)
    │
    ▼
api-middleware (FastAPI)
    │
    ├── 1. 意图识别 (IntentRecognizer)
    │       ├── 提取股票代码
    │       ├── 识别实时数据需求
    │       ├── 识别历史数据需求
    │       ├── 识别技术指标需求 (MACD/BOLL/RSI/MA)
    │       └── 识别基本面数据需求 (财务指标/报表)
    │
    ├── 2. 数据查询
    │       ├── 实时数据 → akshare API
    │       ├── 历史K线 → PostgreSQL stock_daily
    │       ├── 技术指标 → PostgreSQL + 实时计算
    │       ├── 财务报表 → PostgreSQL stock_financial_report
    │       └── 财务指标 → PostgreSQL stock_financial_indicator
    │
    ├── 3. Prompt构建 (build_prompt)
    │       └── 将所有数据整合到System Prompt
    │
    └── 4. 转发到vLLM
            └── vLLM基于数据生成分析回答
```

## 🔍 意图识别关键词

### 股票K线数据关键词
- **历史数据**: "历史走势", "K线", "技术分析", "趋势", "前期", "近一个月", "近三个月"
- **实时数据**: "今天", "今日", "现在", "实时", "当前", "盘中", "最新"

### 技术指标关键词

| 指标 | 触发关键词 | 计算方式 | 返回数据 |
|------|-----------|---------|---------|
| **MACD** | "MACD", "金叉", "死叉" | EMA(12)-EMA(26) | MACD值、Signal值、Histogram、金叉死叉信号 |
| **BOLL** | "BOLL", "布林带", "上轨", "中轨", "下轨" | MA(20)±2σ | 上/中/下轨、带宽、当前位置 |
| **RSI** | "RSI", "超买", "超卖", "相对强弱" | 14日价格变动 | RSI值、超买超卖信号 |
| **MA** | "MA", "均线", "五日线", "十日线", "MA5", "MA10", "MA20", "MA60" | 简单移动平均 | MA5/10/20/60数值、均线排列判断 |

### 基本面数据关键词

| 数据类型 | 触发关键词 | 数据来源 | 返回字段 |
|---------|-----------|---------|---------|
| **财务报表** | "财报", "财务报表", "资产负债表", "利润表", "年报", "季报" | stock_financial_report | 总资产、负债、营收、净利润、现金流 |
| **财务指标** | "ROE", "ROA", "毛利率", "净利率", "EPS", "市盈率", "财务指标" | stock_financial_indicator | ROE、ROA、毛利率、净利率、每股收益、PE、PB |
| **盈利能力** | "盈利能力", "盈利", "营收", "净利润", "毛利" | stock_financial_indicator | ROE趋势、毛利率趋势、净利率趋势 |
| **偿债能力** | "偿债能力", "负债", "资产负债率", "流动比率" | stock_financial_indicator | 资产负债率、流动比率、速动比率趋势 |

## 🔌 API端点列表

### 数据查询API

| 端点 | 方法 | 描述 | 数据来源 |
|------|------|------|----------|
| `/api/stock/{code}/realtime` | GET | 实时行情 | akshare API |
| `/api/stock/{code}/history` | GET | 历史K线 | PostgreSQL stock_daily |
| `/api/stock/{code}/indicators` | GET | 技术指标 | PostgreSQL + 实时计算 |
| `/api/stock/{code}/indicators/analysis` | GET | 多指标综合分析 | PostgreSQL + 实时计算 |

| `/api/stock/{code}/financial/summary` | GET | 财务摘要 | PostgreSQL |
| `/api/stock/{code}/full-analysis` | GET | 完整分析（技术+基本面+实时） | 综合 |

| 端点 | 方法 | 描述 | 数据整合 |

## 📝 数据表结构

### 1. stock_daily（历史K线数据）
```sql
- code: 股票代码
- trade_date: 交易日期
- open, high, low, close: OHLC价格
- volume, amount: 成交量额
- change_pct: 涨跌幅
- turnover_rate: 换手率
- amplitude: 振幅
```

**用途**: 计算技术指标（MA/MACD/BOLL/RSI）、K线走势分析

### 2. stock_financial_report（财务报表）
```sql
- code: 股票代码
- report_date: 报告日期
- report_type: 报表类型（balance_sheet/income_statement/cash_flow）
- total_assets, total_liabilities: 资产负债
- total_revenue, net_profit: 营收利润
- cash_and_equivalents: 货币资金
- 等20+财务字段
```

**用途**: 基本面分析、财务健康度评估

### 3. stock_financial_indicator（财务指标）
```sql
- code: 股票代码
- report_date: 报告日期
- eps_basic, bps: 每股收益/净资产
- roe_weighted, roa: ROE/ROA
- net_profit_margin, gross_profit_margin: 净利率/毛利率
- debt_asset_ratio, current_ratio: 资产负债率/流动比率
- 等30+财务指标
```

**用途**: 盈利能力分析、偿债能力分析、估值分析

## 🔧 技术指标计算说明

### MACD计算
```python
EMA(12) = 前一日EMA(12)×11/13 + 今日收盘价×2/13
EMA(26) = 前一日EMA(26)×25/27 + 今日收盘价×2/27
DIF = EMA(12) - EMA(26)
DEA = 前一日DEA×8/10 + 今日DIF×2/10
MACD = (DIF - DEA) × 2
```

### BOLL计算
```python
中轨 = MA(20)
标准差 = √[Σ(收盘价-MA)^2 / 20]
上轨 = 中轨 + 2×标准差
下轨 = 中轨 - 2×标准差
```

### RSI计算
```python
RS = 14日平均上涨幅度 / 14日平均下跌幅度
RSI = 100 - 100/(1+RS)
```

## 🎯 使用建议

1. **技术分析场景**: 使用关键词"MACD"、"布林带"、"均线"、"技术指标"
2. **基本面分析场景**: 使用关键词"ROE"、"财务指标"、"盈利能力"、"基本面"
3. **综合分析场景**: 同时包含技术面和基本面关键词
4. **实时行情场景**: 使用关键词"今天"、"现在"、"最新"

通过精心设计用户提问，可以触发系统自动获取相应的数据并注入到prompt中，让vLLM生成基于真实数据的分析回答。
