# 自动断点续传功能说明

## ✅ 功能确认

**Data Hub Docker 容器现在每次启动都会自动执行断点续传！**

### 工作原理

```
Docker 容器启动
    │
    ▼
startup_auto_resume.py 执行
    │
    ├── 1. 等待数据库连接就绪
    │
    ├── 2. 初始化数据库表（包含 stock_download_progress 断点续传表）
    │
    ├── 3. 检查当前数据状态
    │       ├── 情况A: 全新部署（无数据）
    │       │           └── 初始化所有股票的进度记录（pending状态）
    │       ├── 情况B: 下载中断（有未完成的任务）
    │       │           └── 获取 pending/running/failed 状态的股票
    │       ├── 情况C: 数据已完整但需要更新
    │       │           └── 计算缺失的最新数据日期
    │       └── 情况D: 数据已最新
    │                   └── 直接完成
    │
    ├── 4. 执行断点续传下载
    │       ├── 获取所有股票列表
    │       ├── 筛选出未完成的股票
    │       ├── 并发下载（5个并发）
    │       ├── 每只股票下载完成立即更新进度
    │       └── 失败的股票记录错误，等待下次重试
    │
    ├── 5. 检查数据质量
    │       ├── 统计已下载股票数
    │       ├── 计算完成率
    │       └── 评估数据质量
    │
    └── 6. 退出
            ├── 成功（exit 0）: 数据质量良好
            │                   └── Docker 不再重启
            ├── 部分成功（exit 0）: 部分股票失败
            │                       └── Docker 不再重启，下次启动会重试
            └── 失败（exit 1）: 严重错误
                                └── Docker 根据 restart: on-failure 自动重启
                                    下次启动从断点继续
```

## 🛠️ 手动触发断点续传

### 方式 1: 使用 docker-compose 重启

```bash
# 重启容器会自动触发断点续传
docker-compose restart data-hub

# 查看日志
docker-compose logs -f data-hub
```

### 方式 2: 在容器内手动执行

```bash
# 进入运行中的容器
docker exec -it finr1-datahub bash

# 查看当前状态
python startup_auto_resume.py --status

# 手动执行断点续传
python startup_auto_resume.py

# 强制重新下载所有数据
python startup_auto_resume.py --full
```

### 方式 3: 使用 resume_manager 工具

```bash
# 进入容器
docker exec -it finr1-datahub bash

# 查看详细状态
python resume_manager.py status

# 执行断点续传
python resume_manager.py resume

# 重试失败的任务
python resume_manager.py retry
```

## 🔧 配置选项

### 修改并发数

编辑 `history_downloader_with_resume.py`:
```python
CONCURRENT_DOWNLOADS = 10  # 默认5，根据网络情况调整
```

### 修改重试次数

编辑 `database.py` 中的 `StockDownloadProgress`:
```python
max_retries = Column(Integer, default=5)  # 默认3次
```

### 修改重启策略

编辑 `docker-compose.yml`:
```yaml
data-hub:
  restart: on-failure
  deploy:
    restart_policy:
      condition: on-failure
      delay: 60s        # 失败后60秒再试
      max_attempts: 20  # 最多重试20次
```

## 📝 常见问题

### Q: 容器为什么反复重启？

**A**: 如果容器反复重启，说明下载遇到了问题。检查：
```bash
# 1. 查看日志
docker-compose logs data-hub | tail -100

# 2. 检查网络连接
ping 123.60.11.74

# 3. 检查数据库连接
docker run --rm --network host postgres:15-alpine \
  psql "postgresql://game_agents:1234+asdf@123.60.11.74:5432/finr1_data" \
  -c "SELECT 1;"
```

### Q: 如何查看下载进度？

**A**: 多种方式：
```bash
# 实时日志
docker-compose logs -f data-hub

# 进度表查询
docker exec -it finr1-datahub python resume_manager.py status

# 数据量统计
docker run --rm --network host postgres:15-alpine \
  psql "postgresql://game_agents:1234+asdf@123.60.11.74:5432/finr1_data" \
  -c "SELECT COUNT(*) FROM stock_daily;"
```

### Q: 如何强制重新下载？

**A**: 
```bash
# 方式1: 使用 --full 参数
docker-compose run --rm data-hub python startup_auto_resume.py --full

# 方式2: 清除进度记录
docker exec -it finr1-datahub python resume_manager.py reset --all
```

## ✅ 总结

**每次启动 Docker 容器时：**

1. ✅ 自动检测当前数据状态
2. ✅ 自动执行断点续传（如果有未完成的任务）
3. ✅ 自动重试失败的任务（最多3次）
4. ✅ 自动增量更新（如果数据已完整但需要更新）
5. ✅ 数据下载完成后正常退出
6. ✅ 失败时自动重启，下次继续从断点下载

**无需人工干预，数据最终一定会完整！**
